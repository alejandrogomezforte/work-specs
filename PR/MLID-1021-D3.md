# [MLID-1021] Hospital System Agreements - Deliverable 3: Bulk Update

## Summary

This PR delivers the third and final deliverable for the Hospital System Agreements epic (MLID-1021). It introduces the complete Bulk Update workflow allowing administrators to replace the full provider NPI list for a hospital system via CSV file upload or paste input. ## API Layer

### `PUT /api/hospitals/[id]/providers/bulk` — Bulk replace providers (MLID-1545, MLID-1558)

Replaces the entire provider NPI list for a hospital system. The route delegates to the existing `bulkUpdateProviders` service function.

| Param | Type | Location | Description |
|-------|------|----------|-------------|
| `id` | string | URL path | Hospital system ObjectId |
| `providers` | array | Body | Array of `{ npi: string }` objects |

**Response cases:**

| Status | Condition |
|:------:|-----------|
| 200 | Success — returns updated hospital system document |
| 400 | Invalid ObjectId, missing/malformed providers array, invalid NPI format (with `invalidNPIs` list), array exceeds 10,000 entries |
| 401 | Unauthenticated |
| 403 | Insufficient permissions (requires Admin, Auditor, or Corporate) |
| 404 | Hospital system not found or soft-deleted |
| 500 | Unexpected server error |

**Security hardening (MLID-1558):**

- **NPI trimming:** Whitespace-padded NPIs are cleaned before validation, preventing false regex failures
- **Max array size guard:** Rejects requests with more than 10,000 provider entries (400 response) to prevent memory/CPU abuse
- **Field stripping:** Only the `npi` field is forwarded to the service layer, preventing arbitrary data injection into MongoDB
- **Non-object entry handling:** Entries like `null`, numbers, or strings in the providers array return 400 instead of 500

**Validation layers:**

| Layer | Responsibility |
|-------|----------------|
| Route (structural) | Verifies `providers` is an array, each item has `npi` as a string, array size ≤ 10,000 |
| Service (business) | Validates NPI format (`/^\d{10}$/`), deduplicates by NPI, returns specific `invalidNPIs` on failure |

## Shared Utilities

### NPI Validation (`utils/validation/npiValidation.ts`) — MLID-1557

Shared validation utility used by both the BulkUpdateTab component and the API route:

| Function | Description |
|----------|-------------|
| `isValidNpi(npi)` | Validates a single NPI matches `^\d{10}$` format |
| `validateNpiList(npis)` | Validates a list of NPIs, deduplicates, and tracks invalid entries with error reasons and duplicate values |

Returns `{ valid: string[], invalid: { npi, reason }[], duplicates: string[] }`.

### CSV Parser (`utils/csv/parseCsvNpis.ts`) — MLID-1555

Parses CSV files into NPI lists:

| Feature | Detail |
|---------|--------|
| Column support | 1-column (NPI only) and 2-column (NPI + Name) formats |
| Header detection | Optional, case-insensitive "NPI" in first cell |
| NPI extraction | Always from first column |
| Error handling | Rejects 3+ column CSVs with clear error message |

## UI Layer

### Bulk Update Tab (`[id]/components/BulkUpdateTab.tsx`) — MLID-1554, 1555, 1556, 1558

Full-featured Bulk Update tab replacing the D2 placeholder:

**Layout:**
- **Warning banner:** Amber `WarningAmber` icon with explanatory text about replacing the full provider list
- **CSV upload section:** `FileDropzone` component with `CloudUploadOutlined` icon in circular container (46×46, border-radius 48px), drag-and-drop support, CSV-only filter, 5 MB max
- **Paste NPI section:** MUI `TextField` multiline with "Paste NPI List (from Excel)" label, handles all line endings (CRLF, CR, LF)
- **Action buttons:** Cancel (left-aligned, secondary) and Confirm & Replace (right-aligned, primary)

**Input handling:**
- **Mutual exclusivity:** FileDropzone is disabled when textarea has content and vice versa — only one input source is active at a time
- **CSV pipeline:** File read → parse (1-col/2-col detection) → validate → display results
- **Paste pipeline:** Live validation on each keystroke with debounced processing
- **"Upload again" button:** Resets CSV state to allow re-upload

**Validation feedback:**
- **ValidBadge component:** Green badge showing "Found N Valid Entries" (bg `#3A888C`, border-radius 4px) in section header
- **ValidationErrors component:** Displays invalid NPIs with error reasons and duplicate NPI values below inputs
- **Confirm & Replace button:** Only enabled when `valid > 0 AND invalid === 0` (all NPIs must be valid)

**Confirm & Replace flow (MLID-1558):**
- Button click calls `bulkReplaceProviders` with hospital ID and valid NPIs
- **Loading state:** Button shows "Replacing..." text and is disabled during submission
- **Success:** Shows success toast, calls `onSuccess` callback (refreshes hospital data + switches to Providers tab)
- **Error:** Shows error toast, preserves user input, re-enables button for retry
