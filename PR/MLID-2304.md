# [MLID-2304] Pin react-pdf to 10.1.0 and drop pdfjs-dist override

## Summary

Removes the workspace-wide `pdfjs-dist` override added in [`62d1bd8f`](https://github.com/LocalInfusion/li-call-processor/commit/62d1bd8fd4011482668070aa518f663938327371) (MLID-1442) and pins `react-pdf` to an exact version, which is the actual root cause of the CI/CD `npm install` failure that the override was patching around.

**Branch:** `fix/MLID-2304-pin-react-pdf-remove-pdfjs-override` → `develop`

**Jira ticket:** MLID-2304

---

## Root Cause

`apps/web/package.json` declared `react-pdf: "^10.1.0"`. When upstream published `react-pdf@10.4.1`, npm satisfied the caret range with the new minor, which transitively pulled in `pdfjs-dist@5.4.296`. That newer pdfjs-dist hit the `Object.defineProperty` webpack collision (webpack/webpack#20095) and broke `npm install` on CI.

MLID-1442 patched the symptom by pinning `pdfjs-dist@5.3.93` via a root-level `overrides` block. The override is no longer needed once `react-pdf` itself is locked.

---

## Fix

- `package.json` — remove `overrides.pdfjs-dist`
- `apps/web/package.json` — `react-pdf: "^10.1.0"` → `"10.1.0"`
- `package-lock.json` — regenerated; `pdfjs-dist@5.3.93` is now resolved naturally as a transitive dep of `react-pdf@10.1.0` (which declares it as an exact dependency)
