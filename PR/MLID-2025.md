# [MLID-2025] feat(orders-tracker): redesign filter header and improve custom filter visibility

## Summary
- Restructure the Order Tracker page header layout: search bar on the left, "Quick Filters:" label, quick filter chips, "Clear Filters" as plain text, then column visibility and filter icons on the right
- Improve custom filter visibility: highlight the filter icon when active, change "Clear Filters" to red "Clear all filters" text, render active custom filters as removable chips below the header
- Fix multi-select filter overlap bug: allow MUI DataGrid's "is any of" Autocomplete input to grow vertically so all selected chips remain visible
- Adopt design system Chip variant/color props (`variant="soft"` + `color="success"`) instead of hardcoded color styles for consistency with `@repo/ui`
- Format date filter values as `M/D/YYYY` and map operator codes to readable labels (`onOrAfter` → "after", `not` → "is not", etc.)
- Truncate multi-value filter chip labels to first 2 values + `+N more` to keep chips compact
- Standardize Chip delete icon size at the design system level (`iconPx`: 16px for `sm`, 20px for `md`) — was inheriting MUI's oversized 22px default

## Root Cause
Multi-select filter overlap: MUI DataGrid Pro's built-in filter panel uses an `Autocomplete` for `singleSelect` columns with the "is any of" operator. The Autocomplete input root has a fixed compact height that clips chips when multiple values are selected. Filtering by 6+ statuses caused chips to pile up and become unreadable.

## What Changed and Why

### 1. Filter header layout restructure (`page.tsx`)

**Before:**
```
[Quick Filters: label] [Clear Filters chip] [QF chips] [Search──] [icons]
```

**After:**
```
[Search──────────] Quick Filters: [QF chips] Clear Filters [icons]
```

- Search `TextField` moved to the left as the first element with `Layout.Item grow="1"`
- New `Typography` label "Quick Filters:" (color `#6B6B74`) sits between the search and the chips
- Quick filter chips moved into their own `Layout direction="row" gap={3}`
- "Clear Filters" was a `Chip onClick`, now a clickable `<span role="button" tabIndex={0}>` wrapping a `Typography` (with `onKeyDown` for Enter/Space accessibility)

### 2. Custom filter visibility (`page.tsx`)

Added page state derived from the existing `filterModel` URL state:
- `hasActiveCustomFilter = filterModel.items.length > 0`
- `activeColumns` — resolves the current tab's column definitions for `headerName` lookup
- `removeFilterItem(index)` — removes a single item from `filterModel.items`, also clears `activeQuickFilter`
- `formatFilterChipLabel(item)` — builds a human-readable label from `field`, `operator`, `value`

Visual changes:
- Filter `IconButton` gets `style={{ color: '#00695C' }}` (teal) when `hasActiveCustomFilter`
- "Clear Filters" → "Clear all filters" with `color: '#D32F2F'` (Material red) when active
- New `Layout direction="row" justify="end" wrap="wrap"` row below the header, rendered only when there are active filters. One `Chip` per filter item with `onDelete` wired to `removeFilterItem`.

### 3. Multi-select filter chips vertical expansion (`DataGrid.styles.ts`)

The MUI DataGrid Pro filter panel renders its own `Autocomplete` for `singleSelect` columns. We can't replace it without owning the slot — but we can target it via CSS:

```ts
'& .MuiInputBase-root.MuiAutocomplete-inputRoot': {
  flexWrap: 'wrap',
  height: 'auto',
  minHeight: 40,
},
```

`flexWrap` lets chips wrap horizontally, `height: auto` removes the fixed compact height, `minHeight: 40` keeps the single-line minimum. No `maxHeight` so the input grows freely to show all chips.

**Alternatives considered:** custom `InputComponent` via column `filterOperators` override (too invasive for the gain), or replacing `baseAutocomplete` slot globally (would affect every Autocomplete in every DataGrid).

### 4. Design system Chip variants (`page.tsx`)

Replace hardcoded styles with design system props that resolve to the same colors via `Chip.styles.ts:36`:

**Before:**
```tsx
style={{ borderColor: '#3A888C', color: '#2E696D', backgroundColor: '#EDF4F1' }}
```

**After:**
```tsx
variant="soft"
color="success"
```

- Quick filter chips: `variant={active ? 'soft' : 'outlined'}` + `color={active ? 'success' : undefined}`
- Active custom filter chips: always `variant="soft"` + `color="success"`

### 5. Date and operator formatting (`page.tsx`)

`formatFilterChipLabel` now produces clean labels:

| Raw | Display |
|-----|---------|
| `lastUpdated`, `onOrAfter`, `2026-04-02T06:00:00.000Z` | `lastUpdated after "4/2/2026"` |
| `status`, `not`, `Scheduled` | `status is not "Scheduled"` |
| `dueDate`, `onOrBefore`, `2026-04-02T06:00:00.000Z` | `dueDate before "4/2/2026"` |

Operator label map: `isAnyOf` → "is any of", `not` → "is not", `onOrAfter`/`after` → "after", `onOrBefore`/`before` → "before".

`formatDateIfISO(value)`:
- Detects strings starting with `YYYY-MM-DD`
- Parses via `new Date()` and formats as `M/D/YYYY`
- Safe fallback to raw value on `isNaN(d.getTime())` or non-date strings (so `"Not Started"` and `"2026-99-99"` pass through unchanged)

### 6. Multi-value chip truncation (`page.tsx`)

When an array filter (e.g., "Order Status is any of [...]") has more than 2 values, show only the first 2 followed by `+N more`:

```
Order Status is any of "Not Started, Working this Week +4 more"
```

Also filters out empty strings and `null` from the array first — fixes a regression where the `activeOrdersByDueDate` quick filter seeds `value` with a leading `undefined`, which produced labels like `", Not Started +5 more"`.

### 7. Chip delete icon size (`Chip.styles.ts`)

MUI's default `CancelIcon` for `MuiChip-deleteIcon` is 22px. On a `sm` chip (24px tall), that filled almost the entire height and looked unbalanced. Added:

```ts
'& .MuiChip-deleteIcon': {
  fontSize: iconPx,  // 16 for sm, 20 for md — matches the left-side icon
},
```

**Note in code:** `li-ui-playground` uses a hardcoded 14px `<CancelIcon style={{ fontSize: 14 }} />` per-instance. We chose `iconPx` for design system consistency across all `Chip` consumers instead of patching individual call sites.
