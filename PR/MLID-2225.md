# [MLID-2225] feat(filter-persistence): persist filter/sort/search state and column order across sessions

## Summary

Persists the orders-tracker and intakes filter/sort/search state across browser sessions using `localStorage`, plus column drag-order on the orders-tracker. State is restored on next page entry — **storage wins over URL** on every mount, so the user gets back exactly what they had.

The implementation went through one significant in-PR refactor: the original page-level `usePersistedRouteQuery` + `useGridUrlModels` rebuilt the entire URL query string from scratch on every state change, which made the writers race each other and risked silently dropping any URL param outside their vocabulary. Replaced with a **three-layer architecture** (storage primitives → URL manager → domain hooks) where each writer patches only the keys it owns.

**Branch:** `feature/MLID-2225-persist-filters-column-order` → `develop`

**Jira:** [MLID-2225](https://localinfusion.atlassian.net/browse/MLID-2225)

## Architecture

```
LocalStorage  ◄── source of truth across sessions
     │
     ▼ (seed once on mount, before any other URL writes)
URL params (Map<key, value>)
     │
     ▼ (parse via useMemo or useState + sync useEffect)
UI components (DataGrid, search box, filter chips)
     │
     ▲ (user change)
     │
URL params  ── set/delete only the keys you own
     │
     ▼ (mirror on every change)
LocalStorage
```

Three modules, each with one job:

### Layer 1 — Storage primitives (`apps/web/utils/hooks/urlPersistence.ts`)

Pure module-level functions. Versioned envelope `{ version: 1, queryString }`.

```ts
export const loadStoredQueryString = (key: string): string | null;
export const saveStoredQueryString = (key: string, queryString: string): void;
export const clearStoredQueryString = (key: string): void;
```

Malformed JSON or wrong version → silently returns `null`. All `localStorage` exceptions (private browsing, quota exceeded) are swallowed.

### Layer 2 — URL manager (`apps/web/utils/hooks/useUrlParams.ts`)

Generic. Knows nothing about sort/filter/pagination/etc.

- Constructor: `useUrlParams(storageKey?: string)`. When `storageKey` is provided, persistence is enabled.
- **Seed on mount** (`hasSeededRef`-gated): if storage has a saved QS that differs from the current URL, calls `window.history.replaceState` **synchronously** plus `router.replace`. The synchronous `replaceState` is essential — Next.js `router.replace` is deferred and any code reading `window.location` inside the same microtask flush would otherwise see the pre-seed URL.
- **Mirror after seed**: every observed `searchParams` change writes the URL back to storage. Reads from `window.location` so the very first mirror tick after seed sees the seeded value (the React-cycle `searchParams` lags one render).
- `setParams(patch: Record<string, string | null>)`: granular merge. `null` deletes a key, string sets it. Other keys pass through untouched. Multiple synchronous calls batch into a single microtask-flushed `router.replace`.
- `clearOwnedKeys(keys: string[])` and `getQueryString()` are convenience wrappers.

### Layer 3 — Domain hooks (refactored)

- `useGridUrlModels(storageKey?: string)` — typed accessors for the orders-tracker's six URL keys (`page`, `pageSize`, `sort`, `filter`, `search`, `quickFilter`). Each setter patches one key. Derived state via `useMemo(searchParams)` auto-resyncs on URL changes.
- `useIntakesTable({ storageKey, initialLocations })` — same model. Local `useState` for everything URL-backed plus per-key sync `useEffect`s that update local state when the corresponding URL string changes externally (e.g., after seed). Search and filter debounces have skip-first-tick guards so the empty mount-state can't be written back to the URL on a 300ms timer and clobber the seed.

### Bonus — Column-order persistence (`Orders.tsx`)

Per-tab keys (`ordersTrackerColumnOrder-{new|maintenance|lead}`). Read once via `useState` initializer (MUI X `DataGrid.initialState.columns.orderedFields` only honors the first render), written on `onColumnOrderChange` via `gridApiRef.current.getAllColumns().map(c => c.field)`.

## Decisions

1. **Storage wins over URL on every entry.** Bookmarked or shared URLs with explicit filters get overwritten by the user's saved state. Trade-off accepted with the user — pasted URLs aren't a primary collaboration channel here.
2. **Granular per-key writes.** No more rebuilding the whole QS. `useGridUrlModels`'s old `buildQs` / `liveStateFromUrl` / `stateRef` scaffolding (~165 lines) is gone — each setter patches its own key on a fresh `URLSearchParams(window.location.search)` and writes back. Future params from unrelated features survive automatically.
3. **`history.replaceState` runs synchronously alongside `router.replace`.** Required so the orders-tracker's microtask flush in `useGridUrlModels` (and any future similar consumer) sees the seeded URL within the same tick.
4. **DataGrid mount-emit guard on the orders-tracker.** MUI X's `useGridFilter` mount-time normalization can fire `onFilterModelChange` with URL-restored items silently dropped (root cause unclear; reproduces with valid singleSelect values across multiple commit cycles, so the existing `hasMounted` ref is too eager). `handleFilterModelChange` now rejects emits where `m.items.length < filterModel.items.length` — that's normalization noise, not user intent. User-initiated removals happen via the active-filter chip's `onDelete`, which bypasses this handler.
5. **Intakes uses local state + per-key sync `useEffect`s** rather than `useMemo`-derived state. Pragmatic choice to keep the existing test surface and the typing UX (search input, filter input) without a larger refactor of the hook.

## What changed

### New modules

- **`urlPersistence.ts`** — versioned localStorage primitives. 100% covered by `urlPersistence.test.tsx` (15 cases).
- **`useUrlParams.ts`** — generic URL manager with granular `setParams`, microtask flush, seed/mirror lifecycle. Covered by `useUrlParams.test.tsx` (17 cases): granular patching, batching, deletion, seeding logic (storage wins, no double-seed, version mismatch discard), mirror gating.

### Refactored

- **`useGridUrlModels.ts`** — wraps `useUrlParams`. Each setter patches one key. `buildQs`, `liveStateFromUrl`, `stateRef`, `pendingRef` deleted. New `useGridUrlModels.test.tsx` (8 cases) asserts granularity (`setSortModel` doesn't disturb `filter`/`search`, `clearModels` preserves unrelated keys, default values omit their URL param entirely).
- **`useIntakesTable.ts`** — drops local `writeUrl`/`buildIntakesUrlParams` (URL-string rebuilder). Each setter calls `setParams` with a one-key patch. Adds skip-first-tick guards on the two debounce useEffects + per-key sync useEffects that re-hydrate local state when the URL changes externally.

### Page wiring

- **`orders-tracker/[category]/page.tsx`** — drops the `usePersistedRouteQuery` import and direct call. Passes the per-tab storage key into `useGridUrlModels(\`ordersTrackerLastQueryString-${tab}\`)`. Adds the `m.items.length < filterModel.items.length` guard inside `handleFilterModelChange` for the DataGrid mount-emit defense.
- **`intakes/IntakesPageNew.tsx`** — same shape. Passes `storageKey: 'intakesLastQueryString'` into `useIntakesTable({ storageKey })`. `clearAllFilters` no longer needs to call any storage `clear()` — the domain hook's setters write the empty URL and the mirror effect saves the empty QS automatically.

### UI fix

- **`intakes/components/IntakesFilterBar.tsx`** — added `justifyContent: 'flex-end'` to the chip-row flexbox so location/type/advanced filter chips line up against the right edge, matching the orders-tracker chip layout per UI designer spec.

### Removed

- **`usePersistedRouteQuery.ts`** + its test (260 lines). Responsibilities split across `urlPersistence` (storage) and `useUrlParams` (seeding/mirroring). New names match the new architecture.

## Notes for reviewer

- **`usePersistedRouteQuery` is gone.** Anything that used to import it now imports `useUrlParams` (URL manager) or the storage primitives directly. (The file pair was created + deleted on this branch, so it doesn't appear in the GitHub diff — see the note above the commits table.)
- **No dedicated test for the orders-tracker DataGrid mount-emit guard.** The `handleFilterModelChange` reject-strict-subset condition in `[category]/page.tsx` was diagnosed and verified manually via console traces (filter went from 2 items → 1 item on hard refresh without the guard, survives with it). The existing `[category]/page.test.tsx` mocks `useGridUrlModels` wholesale, so simulating MUI X's mount-time `onFilterModelChange` cascade through the real component tree is non-trivial. The behavioral surface is tested at the user-action level (chip removal still works); the mount-emit guard itself relies on manual verification. A targeted test using a partial DataGrid mock that captures `onFilterModelChange` would close this gap — flagging as a follow-up.
- **DataGrid filter panel's per-row trash icon won't remove items on the orders-tracker** — only the active-filter chip's `onDelete` works. This is the trade-off of the mount-emit guard (we can't distinguish mount-time normalization from user-initiated panel removal at the handler level). The chip-removal UX is the primary path users already see, and chips are always rendered when filters are active.
- **Intakes uses `useState` + per-key sync `useEffect`s** rather than `useMemo`-derived state (the orders-tracker pattern). Smaller delta vs. the original implementation, and keeps the typing UX for the search input. If the team prefers full convergence on the `useMemo` pattern, that's a clean follow-up.
- **The `tabParamsCache` module-level variable in `orders-tracker/[category]/page.tsx` survives** — it's the in-session per-tab cache used during tab switches, separate from the cross-session storage layer. Both coexist intentionally.
