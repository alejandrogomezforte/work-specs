# [MLID-2213] feat(orders-filter): "(No Value)" option for every enum filter dropdown on the orders tracker

## Summary

Extends every `singleSelect` (enum) column on the orders tracker so the filter dropdown shows a `(No Value)` option that filters to rows where the field is missing, `null`, or `''`. The option appears **only in the filter UI** — never in the inline-edit dropdown — by adopting MUI X v8's function-form `valueOptions`. The server translates a sentinel string into the existing `emptyPredicate` / `notEmptyPredicate` helpers in `gridToMongoQuery.ts`.

**No new MUI X operators** are introduced; we only widen the value list, per the PO request.

**Branch:** `feature/MLID-2213-enum-no-value-filter` → `develop`

**Jira:** [MLID-2213](https://localinfusion.atlassian.net/browse/MLID-2213)

## What changed

### Server (filter translator)

`apps/web/utils/api/orders/gridToMongoQuery.ts` exports a new constant:

```ts
export const NO_VALUE_SENTINEL = '__LI_NO_VALUE__';
```

The translator substitutes the sentinel for `singleSelect` filters:

| Operator | Sentinel substitution |
|----------|----------------------|
| `is` / `equals` / `=` | `emptyPredicate(field)` → `{ $or: [ {field: {$exists: false}}, {field: null}, {field: ''} ] }` |
| `not` / `doesNotEqual` / `!=` | `notEmptyPredicate(field)` → `{ $and: [ {field: {$exists: true}}, {field: {$ne: null}}, {field: {$ne: ''}} ] }` |
| `isAnyOf` (only the sentinel) | `emptyPredicate(field)` |
| `isAnyOf` (sentinel + real values) | `{ $or: [ {field: {$in: [...real]}}, emptyPredicate(field) ] }` |

Strict equality on a server-defined constant prevents arbitrary client-supplied strings from triggering the empty-predicate path.

### Client (filter-only enum option)

New helper at `apps/web/app/orders-tracker/utils/enumValueOptions.ts`:

```ts
export const NO_VALUE_LABEL = '(No Value)';

export function withNoValueOption<T extends string | number>(
  enumValues: readonly T[]
): (params: { field: string; id?: unknown; row?: unknown }) => ValueOptions[];
```

Using the MUI X v8 contract: `valueOptions` is invoked **without `row`** for the filter panel and **with `row`** for inline edit. The helper branches on `params.row` to expose or hide the sentinel option — no defensive `valueSetter` needed.

This is the **first usage of MUI X v8 function-form `valueOptions` in `apps/web/`** — flagged for reviewer awareness.

### Wiring

The 10 affected enum columns have their `valueOptions: Object.values(SomeEnum)` replaced with `valueOptions: withNoValueOption(Object.values(SomeEnum))`:

- **New tab** (`NewOrders.tsx`) — `orderType`, `coverageStatus`, `status`, `liPharmacyStatus`, `audit`
- **Maintenance tab** (`MaintenanceOrders.tsx`) — `changeType`, `coverageStatus`, `status`
- **Lead tab** (`LeadOrder.tsx`) — `orderType`, `status`

(`coverageStatus`, `status`, `liPharmacyStatus`, and `audit` on the new/maintenance tabs render only when the `ORDERS_TRACKER_IG_COLUMNS` flag is on.)

### Chip label

The inline `formatFilterChipLabel` in `[category]/page.tsx` maps the sentinel to `NO_VALUE_LABEL` so users see `Order Status is "(No Value)"` instead of `__LI_NO_VALUE__`. Substitution happens on both the single-value path and the `isAnyOf` array path (BEFORE truncation, so the sentinel counts toward visible/remaining like any other value).

### `audit` / `Audit.NA` semantics

By design, `(No Value)` does **not** match rows with `audit === 'N/A'`. Those rows render blank via `valueFormatter` but the underlying value is the string `'N/A'`, not null. Matching `'N/A'` would couple `gridToMongoQuery` to a domain enum; flagged as an Open Question in the plan if the PO disagrees.
