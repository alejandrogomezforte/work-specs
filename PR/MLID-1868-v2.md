# [MLID-1868] feat(intakes): show skip reason in Status column with expandable text

## Summary

- Restore "Errors" column to original behavior (errors only, no skip reason)
- Move skip reason display to the Status column, shown below the status badge for skipped intakes
- Skip reason rendered in lighter color and smaller font with truncation and "show more"/"show less" toggle
- Add min-width to "Created At" and "Patient DOB" columns to prevent date text from wrapping when the Status column expands

## What Changed and Why

### 1. Errors column restored (`page.tsx`)

PO feedback: the "Note" column rename was confusing — the column should remain "Errors" and only show `ErrorIndicator` as before. Skip reason was removed from this column entirely.

### 2. Skip reason in Status column (`page.tsx`)

For skipped intakes, the `skipReason` text now renders below the status badge inside an `ExpandableText` component:
- Lighter color (`#71717a`) and smaller font (`0.75rem`) to visually separate it from the status
- Truncated to 1 line via `-webkit-line-clamp`
- "show more" / "show less" toggle link appears when text overflows
- Click events are stopped so expanding/collapsing doesn't trigger row navigation

### 3. Date column min-widths (`page.tsx`)

"Created At" and "Patient DOB" headers get `minWidth: 150px` to prevent date values from breaking into multiple lines when the Status column expands with long skip reasons.

### 4. Updated tests (`page.test.tsx`)

- **Errors column (restored)**: verifies "Errors" header, ErrorIndicator renders for errors, no skipReason leaks into Errors column
- **Skip reason in Status column**: verifies skipReason appears for skipped intakes, expandable structure via `data-testid`, no skipReason for non-skipped intakes
