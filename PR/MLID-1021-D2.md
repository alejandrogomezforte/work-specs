## API Layer

### `GET /api/hospitals/[id]/providers` ŌĆö List providers for a hospital (MLID-1546)

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `page` | number | 1 | Page number (min: 1) |
| `pageSize` | number | 25 | Results per page (min: 1, max: 100) |
| `search` | string | ŌĆö | Filter by NPI or provider name |

**Response:** `{ data: ProviderItem[], pagination: { page, pageSize, totalCount, totalPages } }`

**Access control:** Same as all hospital endpoints ŌĆö requires Admin, Auditor, or Corporate role.

**Input sanitization:** Query params are clamped to valid ranges (negative page ŌåÆ 1, pageSize capped at 100, non-numeric values fall back to defaults). This sanitization was also backported to `GET /api/hospitals` (list endpoint from D1).

## UI Layer

### Hospital Detail Page (`app/hospital-system-agreements/[id]/page.tsx`) ŌĆö MLID-1551

Full detail page accessible by clicking a hospital name in the list:

- **Breadcrumb:** "ŌåÉ Hospital System Agreements" link back to list page
- **Header card:** Hospital name, provider count, last updated date + user
- **"Edit Hospital" button** in header to open the edit modal
- **Tab bar:** "Providers" and "Bulk Update" tabs (Bulk Update is a placeholder for D3)
- **Permission guard:** Redirects to home if user lacks `canManageHospitalSystems` permission
- **Error states:** Dedicated views for loading, not found (404), and generic errors

### Edit Hospital Modal (`components/AddEditHospitalModal.tsx`) ŌĆö MLID-1552

Refactored the D1 `AddNewHospitalModal` into a shared **dual-mode** `AddEditHospitalModal` component:

| Mode | Behavior |
|------|----------|
| `add` | Empty name field, calls `createHospitalSystem()`, shows "Add" / "Adding..." |
| `edit` | Pre-filled name, calls `updateHospitalSystem()`, shows "Save" / "Saving..." |

- Accepts `mode`, `hospitalId`, and `initialName` props
- Same validation rules: empty/whitespace rejected, max 1024 chars, duplicate name (409) shown inline
- All existing add-mode tests updated, new edit-mode tests added
- D1 list page updated to use `<AddEditHospitalModal mode="add" />` (no behavior change)

### Providers Tab (`[id]/components/ProvidersTab.tsx`) ŌĆö MLID-1553

NPI provider table rendered inside the Providers tab:

- **Table columns:** NPI, Provider Name (shows "---" for empty/undefined names)
- **Search:** Text input with `SearchIcon`, 500ms debounce
- **Pagination:** Reuses shared `Pagination` component with page size selector (10/25/50/100)
- **URL persistence:** Page, pageSize, and search params are synced to URL query params so state survives page refresh and browser back/forward
- **States:** Loading spinner, error message, empty state ("No providers found.")
- **Fallback pagination bar:** When results fit in a single page, shows a simplified "Showing X ŌĆō Y of Z results" bar with page size selector

### Client Service Additions (`services/hospitalSystems/hospitalSystemService.ts`)

Three new client-side functions added in D2:

| Function | Description |
|----------|-------------|
| `fetchHospitalSystemById(id)` | GET `/api/hospitals/{id}`, throws `HospitalNotFoundError` on 404 |
| `updateHospitalSystem(id, name)` | PUT `/api/hospitals/{id}`, parses API error messages |
| `fetchProviders(hospitalId, params)` | GET `/api/hospitals/{id}/providers` with pagination + search |

Also added types: `HospitalSystemDetail`, `ProviderItem`, `ProvidersListResponse`, `HospitalNotFoundError`.

## Feature Flag (`HOSPITAL_SYSTEMS`) ŌĆö MLID-1741

New feature flag to control visibility of the Hospital Systems sidebar nav entry:

| Property | Value |
|----------|-------|
| Flag name | `HOSPITAL_SYSTEMS` |
| Code default | `false` (disabled) |
| When OFF | Nav entry is hidden, even for users with permission |
| When ON | Nav entry is shown for users passing `canManageHospitalSystems()` check |
| Toggle | Administration ŌåÆ Feature Flags (no code change needed) |

**Implementation:** Added `HOSPITAL_SYSTEMS` to `FeatureFlag` enum and `DEFAULT_FEATURE_FLAGS` in `featureFlags.ts`. Sidebar conditionally renders the link with `isHospitalSystemsEnabled && userObj && canManageHospitalSystems(userObj)`.
