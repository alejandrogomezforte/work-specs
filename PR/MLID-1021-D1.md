## Data Layer

### MongoDB Schema (`models/HospitalSystem.ts`)

New `hospitalSystems` collection with the following fields:

| Field | Type | Details |
|-------|------|---------|
| `name` | String | Required, max 1024 chars, text-indexed for search |
| `providerNPIs` | String[] | Array of 10-digit NPI strings, validated with `/^\d{10}$/` |
| `createdBy` | String | User email who created the record |
| `updatedBy` | String | User email who last updated the record |
| `deletedAt` | Date | Soft-delete timestamp (null = active) |
| `providerCount` | Virtual | Computed from `providerNPIs.length` |

**Indexes:** text index on `name`, index on `deletedAt`.

### Server Service Layer (`services/mongodb/hospitalSystem.ts`)

Seven service functions for database operations:

| Function | Description |
|----------|-------------|
| `getHospitalSystems()` | Paginated list with search (regex + text), sorting, soft-delete filtering |
| `getHospitalSystemById()` | Single record lookup by ID |
| `createHospitalSystem()` | Create with `createdBy`/`updatedBy` audit fields |
| `updateHospitalSystem()` | Update name with `updatedBy` audit field |
| `deleteHospitalSystem()` | Soft delete (sets `deletedAt`) |
| `getProviders()` | Retrieve provider NPIs for a hospital system |
| `bulkUpdateProviders()` | Replace all provider NPIs with a new array |

## API Layer

### `GET /api/hospitals` — List hospital systems

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `page` | number | 1 | Page number |
| `pageSize` | number | 25 | Results per page |
| `search` | string | "" | Search term (name) |
| `sortBy` | string | "name" | Sort field |
| `sortOrder` | string | "asc" | Sort direction |

**Response:** `{ data: [...], pagination: { page, pageSize, totalCount, totalPages } }`

### `POST /api/hospitals` — Create hospital system

**Body:** `{ name: string }`

**Validations:**
- `typeof name !== 'string'` type guard (prevents NoSQL injection via operator objects)
- Empty/whitespace-only name rejected (400)
- Name exceeding 1024 characters rejected (400)
- Duplicate name returns 409

**Response:** `{ data: { ...hospitalSystem } }` (201)

### `GET /api/hospitals/[id]` — Get hospital system by ID

**Response:** `{ data: { ...hospitalSystem } }`

### `PUT /api/hospitals/[id]` — Update hospital system

**Body:** `{ name: string }`

### `DELETE /api/hospitals/[id]` — Soft-delete hospital system

**Response:** `{ message: "Hospital system deleted successfully" }`

### Access Control

All endpoints require authentication (NextAuth session) and authorization via `canManageHospitalSystems()`. Permitted roles: **Admin**, **Auditor**, and users with **Corporate** position.

## Security Hardening

Three layers of input validation on the hospital name field:

| Layer | Protection |
|-------|-----------|
| **Client** | `inputProps={{ maxLength: 1024 }}` — HTML-level character limit |
| **API Route** | `typeof` type guard rejects non-string payloads (NoSQL injection prevention) + 1024-char max length check (400 response) |
| **Mongoose Schema** | `maxlength: 1024` — database-level constraint |
