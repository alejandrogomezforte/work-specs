## What Changed and Why

### Pharmacy Pipeline Stages (MLID-1861)

Extended the eligibility pipeline function (renamed from `get340BEligibilityStages` to `getEligibilityPipelineStages`) with 2 new stages and a restructured cleanup stage. The function now returns 7 stages total (was 5):

**New stages:**

5. `$lookup: insurance_plans` â€” matches `planName` from the shared `_primaryInsurance.insurance_plan_name`. Returns `isPharmacyEligible` flag. Uses `$limit: 1` defensively.
6. `$addFields: pharmacyEligible` â€” `$switch` computation with 3 outcomes:
   - **No primary insurance** -> `"Waiting on Insurance Input"`
   - **All conditions met** -> `"Yes"`:
     - `drug.pharmacyEligible` is NOT explicitly `false` (true/null/undefined all pass)
     - `site.isPharmacyEligible` = true
     - Insurance criteria: Medicare/Medicare Advantage auto-pass, OR Commercial/Medicaid with `insurancePlan.isPharmacyEligible` = true
   - **Otherwise** -> `"No"`

**Modified stages:**

4. `$addFields: hospital340BEligible` â€” Changed `$eq` for "Commercial" to `$regexMatch` with `options: 'i'` for case-insensitive matching
7. `$unset` (was stage 5) â€” Moved to the end so pharmacy stages can reuse `_primaryInsurance`. Now cleans up `_insurancePlan` in addition to `_providerNpi`, `_340b_hospitals`, `_primaryInsurance`.

**Case-insensitive payer type matching:**

All `payer_type_name` comparisons now use `$regexMatch` instead of `$eq`/`$in`:

| Comparison | Regex | Matches |
|------------|-------|---------|
| 340B Commercial check | `commercial` / `i` | "Commercial", "commercial", "COMMERCIAL", etc. |
| Pharmacy Medicare auto-pass | `medicare(\s+advantage)?` / `i` | "Medicare", "Medicare Advantage", "medicare advantage", etc. |
| Pharmacy Commercial/Medicaid check | `commercial\|medicaid` / `i` | "Commercial", "Medicaid", "commercial", etc. |

All regex inputs are wrapped with `$ifNull: [..., '']` to safely handle null/missing values.

## Architecture Decisions

| Decision | Rationale |
|----------|-----------|
| **Reuse shared `_primaryInsurance` lookup** | Already added by 340B stages (D1). Moved `$unset` to stage 7 so pharmacy stages can access the shared data. |
| **Case-insensitive regex for payer types** | Protects against inconsistent casing in source data. Also changed the existing 340B Commercial check for consistency. |
| **`drug.pharmacyEligible != false` (not `== true`)** | Only explicitly `false` means ineligible. `true`, `null`, `undefined` all count as eligible â€” matching business rule update from 3/4/2026. |
| **Single spread-ternary for both columns** | Both 340B and Pharmacy columns live in the same conditional array, controlled by one flag. Simpler than two separate conditionals. |
