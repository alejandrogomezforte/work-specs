# [MLID-2096] feat(spruce): archive Skyrizi texts after sending

## Summary
- Add archival support for two Skyrizi SMS text types (`skyrizi-obi-training` and `skyrizi-complete-promotion`) to the Spruce conversation webhook handler
- Both Skyrizi sources now get the same automatic archival behavior as REMINDER, SYMPTOM, and WHAT_TO_EXPECT: an internal "Processed by LISA" note is posted, then the conversation is archived
- This is a config-only extension of the pattern established in MLID-1846 — no changes to the archival function itself

## What Changed and Why

### Config map extension (`spruce_conversation.ts`)

Added 2 entries to `ARCHIVABLE_SOURCE_LOG_PREFIXES`:

```typescript
[MessageSource.SKYRIZI_OBI_TRAINING]: 'SMS Skyrizi OBI Training Archival:',
[MessageSource.SKYRIZI_COMPLETE_PROMOTION]: 'SMS Skyrizi Complete Promotion Archival:',
```

The `archiveSmsConversation` function uses a `$in` query against all keys in this map, so it's fully source-agnostic — no code changes needed beyond the config.

Updated the JSDoc comment to document all 5 archivable sources.

### Tests (`spruce_conversation.test.ts`)

Two new `describe` blocks mirroring the existing WHAT_TO_EXPECT pattern:

1. **SMS Skyrizi OBI Training Archival** (3 tests)
2. **SMS Skyrizi Complete Promotion Archival** (3 tests)

Each block covers:
- Happy path: internal note posted and conversation archived when feature flags enabled
- Flags disabled: neither note nor archive called
- Note failure: archive skipped when note posting throws
