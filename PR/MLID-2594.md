# [MLID-2594] fix(auth-review): make entire Prior Auth table row clickable

## Summary

- In the Order Details → Prior Auth tab (`/orders-tracker/{category}/{orderId}/auth-review`), clicking a row now opens the Auth Review detail no matter which cell you click.
- Previously only the **Received** date cell navigated; Drug, Payor, Letter Type, Auth Valid Period, and Review Status cells did nothing.
- This brings the Order Details Prior Auth table in line with the Patient card → Prior Auth tab, where the whole row is already clickable.

## Root Cause

The Prior Auth table is rendered with the `@repo/ui` `DataGrid`. The only navigation was a Next.js `<Link>` wrapping the content of the `receivedAt` column's `renderCell`. Every other column rendered plain text with no click handler, and the `<DataGrid>` had no `onRowClick` prop — so only the Received cell was interactive.

## What Changed and Why

### 1. Whole-row navigation via `onRowClick`

The `@repo/ui` `DataGrid` already extends `@mui/x-data-grid-pro`'s props and supports `onRowClick` as a first-class, documented prop (it even applies a `LiDataGrid--clickable` pointer-cursor class automatically). No change to the shared `@repo/ui` package was required — we only consume the existing prop.

Before:
```tsx
<DataGrid
  rows={rows}
  columns={columns}
  /* ...pagination props... */
  sx={{ border: 'none', borderRadius: 0 }}
/>
```

After:
```tsx
<DataGrid
  rows={rows}
  columns={columns}
  /* ...pagination props... */
  onRowClick={({ row }) =>
    router.push(`${basePath}/auth-review/${row._id}`)
  }
  sx={{ border: 'none', borderRadius: 0 }}
/>
```

### 2. Removing the Received-cell `<Link>`

Once the entire row navigates, the `<Link>` on the Received cell is redundant and would create a nested-anchor / double-navigation hazard. It is replaced with a plain `<Typography>`, preserving the visual date rendering. The now-unused `next/link` import is removed.

Before:
```tsx
renderCell: ({ row }) => (
  <Link href={`${basePath}/auth-review/${row._id}`}>
    {formatUTCDate(row.receivedAt) || '—'}
  </Link>
),
```

After:
```tsx
renderCell: ({ row }) => (
  <Typography variant="bodySmall">
    {formatUTCDate(row.receivedAt) || '—'}
  </Typography>
),
```

### 3. Preventing the preview button from navigating

The last column's preview `IconButton` opens an in-page document preview panel via state. Because that button lives inside the row, its click would bubble up and also trigger `onRowClick`, navigating away. Adding `event.stopPropagation()` keeps the preview action isolated — matching the existing `PatientPriorAuth` reference pattern.

Before:
```tsx
<IconButton onClick={() => setPreviewLetter(row)}>
```

After:
```tsx
<IconButton
  onClick={(event) => {
    event.stopPropagation();
    setPreviewLetter(row);
  }}
>
```

### Test mock change

The Jest `@repo/ui` mock (`apps/web/mocks/repo-ui.tsx`) is a hand-written stand-in for the real DataGrid and did not forward `onRowClick`. It now wires `onRowClick` onto each rendered row `<div>` and adds a per-row `data-testid`, so the new test can exercise a row click. The change is additive and optional (`onRowClick?.(...)`), so all existing consumers of the mock are unaffected.

## Design Notes

- **No `@repo/ui` change.** The fix consumes the existing `onRowClick` prop on Constantine's shared `DataGrid`; the package source is untouched, so `li-ui-playground` and other consumers are unaffected.
- **Keyboard activation.** Unlike the reference `PatientPriorAuth` (which uses focusable `<tr>` elements with Enter/Space handlers), MUI DataGrid rows are not natively focusable in the same way. `onRowClick` covers mouse interaction; full keyboard row-activation parity is left as a known limitation rather than over-engineered for this bug.
