# [MLID-1780] fix(intakes): support full name search by concatenating firstName + lastName

## Summary

- Full name search now works on the Intakes page (e.g., "Kristen Johnson" finds the correct patient)
- Search input is trimmed to ignore leading/trailing whitespace

## Problem

Searching by full name (e.g., "Kristen Johnson") returned 0 results because `firstName` and `lastName` are stored as separate fields. The regex `"Kristen Johnson"` matched neither `"Kristen"` nor `"Johnson"` individually. Additionally, trailing spaces (e.g., `"kristen    "`) caused the search to take the wrong code path and return no results.

## Fix

- **Multi-word search**: When the search input contains multiple words, use MongoDB `$expr` with `$concat` to build a virtual `"firstName lastName"` field and `$regexMatch` the search string against it. Name order is respected â€” "Johnson Kristen" will NOT match "Kristen Johnson".
- **Single-word search**: Unchanged â€” matches against `firstName` or `lastName` individually via `$or`.
- **Trim**: The search param is trimmed so whitespace-only padding doesn't affect behavior.

### Search behavior summary

| Search input | Matches | Strategy |
|---|---|---|
| `"Kristen"` | Any patient with "Kristen" in firstName or lastName | Single word â†’ `$or` regex on each field |
| `"Kristen Johnson"` | Only "Kristen Johnson" | Multi-word â†’ `$expr` concat `firstName + " " + lastName` |
| `"Johnson Kristen"` | No match for "Kristen Johnson" | Order is respected |
| `"12345"` | Name regex + exact patientId match | Numeric â†’ `$or` includes `patient.patientId` |
| `"kristen    "` | Same as `"kristen"` | Trimmed before processing |
