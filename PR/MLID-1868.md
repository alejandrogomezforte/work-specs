# [MLID-1868] feat(intakes): require skip reason when skipping an intake

## Summary

- Add mandatory skip reason dialog when skipping an intake in the intake analyzer
- Store `skipReason` on the intake MongoDB document alongside `status: 'skipped'`
- Rename "Errors" column to "Note" in intakes list — show skip reason for skipped intakes, error indicator for others
- Display skip reason on intake detail page in the Errors & Warnings card

## What Changed and Why

### 1. Mandatory Skip Reason (`SkipReasonDialog.tsx`)

Previously, skipping an intake was a one-click action with no audit trail. Now a MUI Dialog prompts the user for a reason before the skip proceeds. The Skip button is disabled until a non-whitespace reason is entered. The dialog disables interactions during submission.

### 2. API Validation (`route.ts`)

The PATCH endpoint now enforces `skipReason` when `status === 'skipped'`. If the field is missing or empty after trimming, a 400 is returned. When transitioning away from skipped (e.g., back to `needsReview`), `skipReason` is cleared automatically.

### 3. Column Rename: "Errors" → "Note" (`page.tsx`)

The intakes list column previously labeled "Errors" now shows "Note". For skipped intakes, the cell displays the `skipReason` text. For non-skipped intakes with errors, the existing `ErrorIndicator` component continues to render as before.

### 4. Detail Page Display (`[id]/page.tsx`)

When viewing a skipped intake, the skip reason appears in the Errors & Warnings card with an info alert, providing full visibility into why the intake was skipped.
