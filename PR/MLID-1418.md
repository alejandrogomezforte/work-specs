## Summary

Add `location_type` field to differentiate clinics, pharmacies, and other entries in the PatientLocations collection. All location dropdowns throughout LISA now filter to show only clinic locations, while admins retain full visibility in the Locations management tab.

## What Changed

### 1. Schema & Types
- Added `location_type` field to Location model (`'clinic' | 'pharmacy' | 'other'`, defaults to `'clinic'`)

### 2. Admin UI
- Added Location Type dropdown to Edit Location modal (Users & Locations page)

### 3. API Filtering
- Added optional `?type=` query parameter to `GET /api/locations` and `GET /api/admin/locations`
- Backward compatible - no filter applied when param is omitted

### 4. Frontend Dropdowns
- All location dropdowns now filter by `type: 'clinic'`:
  - `/intakes` - location filter
  - `/intakes/[id]/review` - patient create form
  - `/orders-tracker` - Add Order/Lead dialogs and filter dropdown
  - `/users-locations` - Users tab filter and edit user dialog
- Exception: Locations tab table shows all types for admin management

### 5. Webhook Backfill
- Modified Looker locations webhook to backfill `location_type: 'clinic'` for existing records using MongoDB's `$ifNull` operator
- Preserves manually-set values; only sets default when field is null/undefined
