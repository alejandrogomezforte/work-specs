# [MLID-1780] fix(intakes): enable search by WeInfuse patient ID

## Summary

- Enabled search by WeInfuse patient ID on the Intakes page
- Input is validated as numeric-only before querying `patient.patientId`, preventing injection

## Problem

The search box placeholder says "Search by patient name or ID..." but searching by patient ID did not work. The API route only matched against `patient.firstName` and `patient.lastName` — the `patient.patientId` (WeInfuse ID) field was never included in the search query.

## Fix

When the search input is **numeric-only** (`/^\d+$/`), an exact match on `patient.patientId` is added to the `$or` query alongside name matches. Non-numeric input skips the patient ID search entirely since WeInfuse IDs are always numeric — this also prevents any injection through that field.

## Files Changed

| File | Change |
|------|--------|
| `apps/web/app/api/intakes/route.ts` | Add numeric validation + `patient.patientId` exact match to search `$or` clause |
| `apps/web/app/api/intakes/route.test.ts` | New test file: 8 tests covering patient ID search, numeric validation, ObjectId, and name search |
