## What Changed and Why

### Archival dispatcher (`archiveSmsConversation`)

**Before**: Three sequential copy-paste blocks — one each for REMINDER, SYMPTOM, and (missing) WHAT_TO_EXPECT. Each block did `Message.findOne` by source, logged, and called `noteAndArchiveConversation`.

**After**: An `ARCHIVABLE_SOURCES` config array maps each `MessageSource` to its log prefix. A single `for...of` loop replaces the repeated blocks. Adding a new source is now a one-liner in the array.

The shared helper `noteAndArchiveConversation` is unchanged — it still handles the internal note + archive flow gated by `SPRUCE_SMS_REMINDER_INTERNAL_NOTE` and `SPRUCE_SMS_REMINDER_ARCHIVE` flags.

### Tests

New `describe('SMS What To Expect Archival')` block with 3 tests mirroring the existing Symptom block:
1. Happy path: note posted and conversation archived when flags enabled
2. Flags disabled: neither note nor archive called
3. Note failure: archive skipped when note posting throws

Updated the existing filter assertion test to also verify `Message.findOne` is called with `MessageSource.WHAT_TO_EXPECT`.
