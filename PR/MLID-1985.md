# [MLID-1985] fix(intakes): fix location filter multi-select overflow with chips below filter bar

## Summary
- Fix the Intakes page location filter where selecting 3+ locations caused chips to overflow and overlap inside the constrained input box
- Replace custom `MuiStandaloneMultiSelect` component with `Autocomplete` + `Chip` from `@repo/ui` design system
- Selected locations now render as removable chips in a horizontal row below the filter bar (matching the approved design from li-ui-playground)
- Delete `MuiStandaloneMultiSelect` component (zero remaining consumers)

## Root Cause
`MuiStandaloneMultiSelect` rendered MUI `Chip` components inside the `TextField` input root. With the `.locationFilter` container capped at `max-width: 400px`, selecting 3+ locations caused chips to wrap and visually overlap inside the fixed-height input box.

## What Changed and Why

### Autocomplete replacement (`page.tsx`)

**Before:** Custom `MuiStandaloneMultiSelect` rendered chips inside the input. With 3+ selections in a 400px container, chips overflowed.

**After:** `@repo/ui` `Autocomplete` with:
- `renderTags={() => null}` — suppresses all chips inside the input
- `getOptionLabel` / `isOptionEqualToValue` — works with `{ value, label }` objects directly (no label-to-ID mapping needed)
- `disableCloseOnSelect` — keeps dropdown open for multi-selection

Selected locations render as `@repo/ui` `Chip` components in a `.selectedLocationTags` container below the filter bar, each with an `onDelete` handler that removes the location from filters.

### Design decision
This approach follows the exact pattern prototyped by the designer in `li-ui-playground/src/pages/Intakes.tsx` (lines 638-691), using the same `Autocomplete` + `Chip` component pair from `@localinfusion/ui`.

### Test updates (`page.test.tsx`)
- All 76 renders now wrapped with `LiThemeProvider` (required by `@repo/ui` Emotion styled-components)
- Loading state tests updated: `@repo/ui` Autocomplete shows a disabled input during loading rather than a `CircularProgress` spinner
- 4 new tests validate chips render in `.selectedLocationTags` container, not inside the autocomplete input
