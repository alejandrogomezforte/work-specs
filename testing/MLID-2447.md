## Pre-test State (as of setup)

- Order `0yn` has **zero** notifications, so the badge does not render yet.
- The badge appears only when the order has at least one unread notification whose
  `entities` array contains `{ entityType: 'order', entityId: '6a34840be879c9374538c919' }`.
- "Unread" (for `type: 'user'` notifications) = no matching row in
  `notificationreads` for `(notificationId, targetUserId)`. Fresh seed rows have
  none, so they count immediately.
