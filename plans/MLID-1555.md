# MLID-1555 — Implement CSV File Upload for Bulk NPI Update

## Context

The Bulk Update tab (`BulkUpdateTab.tsx`) currently has a **static UI placeholder** for CSV upload — no drag-and-drop, no file parsing, no validation. This task makes the CSV upload section **and** the paste NPI textarea fully functional: users can drop/select a CSV file or paste NPIs directly, the app parses and validates them, and displays results (valid count, invalid entries with reasons, duplicates).

A reusable `FileDropzone` component already exists at `components/UI/FileDropzone/` with drag-and-drop, size validation, and accept-type filtering.

**Scope boundaries:**
- **In scope:** CSV upload, paste NPI textarea, parsing, NPI validation, client-side validation feedback, Confirm & Replace button state
- **Out of scope:** Confirm & Replace API call (MLID-1558)
- **Dependency:** NPI validation is needed here. We created the shared utility (`npiValidation.ts`) that MLID-1557 describes, since this task requires it. MLID-1557 can be closed or reduced to "refactor existing consumers".

## Implementation Steps

### Step 1 — Create NPI validation utility (TDD)

**New file:** `apps/web/utils/validation/npiValidation.ts`
**New test:** `apps/web/utils/validation/__tests__/npiValidation.test.ts`

```ts
// Types
export interface InvalidNpiEntry {
  value: string;
  error: string;
}

export interface NpiValidationResult {
  valid: string[];
  invalid: InvalidNpiEntry[];
  duplicates: string[];       // distinct NPI values that appeared more than once
  totalInput: number;
  duplicatesRemoved: number;
}

// Functions
export function isValidNpi(npi: string): boolean;          // /^\d{10}$/
export function validateNpiList(rawEntries: string[]): NpiValidationResult;
```

`validateNpiList` flow: trim → filter empties → validate format → deduplicate valid → track duplicate values → return result.

Error messages: "NPI must be exactly 10 digits", "NPI must contain only numeric digits".

### Step 2 — Create CSV parsing utility (TDD)

**New file:** `apps/web/utils/csv/parseCsvNpis.ts`
**New test:** `apps/web/utils/csv/__tests__/parseCsvNpis.test.ts`

```ts
export interface CsvParseResult {
  npis: string[];       // Raw extracted values (pre-validation)
  error?: string;       // File-level error (empty file, unsupported format, etc.)
}

export function parseCsvNpis(csvText: string): CsvParseResult;
```

Supported CSV formats (only these two):
1. **1 column** — NPI values only, header optional (skip if first cell is "npi" case-insensitive)
2. **2 columns** — NPI (first column) + Name (second column), header optional

Any CSV with 3+ columns returns an error: "Unsupported CSV format. File must have 1 column (NPI) or 2 columns (NPI, Name)."

NPI is always extracted from the **first column**. No header-name-based column detection.

### Step 3 — Update BulkUpdateTab component

**Edit:** `apps/web/app/hospital-system-agreements/[id]/components/BulkUpdateTab.tsx`

Changes:
1. Replace static `.uploadZone` div with `<FileDropzone>` component
   - `accept={{ 'text/csv': ['.csv'] }}`
   - `maxSize={5 * 1024 * 1024}` (5MB)
   - `onFilesSelected` handler
2. Add CSV state: `csvFile`, `fileError`, `csvValidation`
3. On file selected: read as text → `parseCsvNpis()` → `validateNpiList()` → set state
4. Show file info after upload (filename)
5. Show validation summary with details:
   - "X valid" / "Y invalid" / "Z duplicates removed" counts
   - List each invalid NPI with its error reason
   - List each duplicated NPI value
6. Add "Upload again" button (resets CSV state and shows FileDropzone again)
7. Disable textarea when CSV is loaded (only one input source at a time)
8. Add paste NPI state: `pasteText`, `pasteValidation`
9. On textarea change: split by newlines → `validateNpiList()` → show same validation summary below textarea
10. Enable "Confirm & Replace" button when either CSV or textarea has valid NPIs
11. Extract `ValidationSummary` as an internal component to share rendering between CSV and textarea sections
