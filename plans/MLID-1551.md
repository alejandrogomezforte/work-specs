## Implementation Plan

### Step 1: Add `fetchHospitalSystemById` to Client Service

**File:** `apps/web/services/hospitalSystems/hospitalSystemService.ts`

Add a new interface and function:

```typescript
export interface HospitalSystemDetail {
  _id: string;
  name: string;
  providerCount: number;
  providerNPIs: string[];
  updatedAt: string;
  updatedBy?: string;
  createdAt?: string;
  createdBy?: string;
}

export const fetchHospitalSystemById = async (
  id: string
): Promise<HospitalSystemDetail> => {
  const response = await fetch(`/api/hospitals/${encodeURIComponent(id)}`);

  if (response.status === 404) {
    throw new HospitalNotFoundError(id);
  }

  if (!response.ok) {
    const errorData = await response.json();
    throw new Error(errorData.error || 'Failed to fetch hospital system');
  }

  const result = await response.json();
  return result.data;
};

export class HospitalNotFoundError extends Error {
  constructor(id: string) {
    super(`Hospital system not found: ${id}`);
    this.name = 'HospitalNotFoundError';
  }
}
```

### Step 2: Create Detail Page Component

**File:** `apps/web/app/hospital-system-agreements/[id]/page.tsx`

```
'use client'

Component: HospitalDetailPage
Props: { params: { id: string } }

State:
  - hospital: HospitalSystemDetail | null
  - isLoading: boolean
  - notFound: boolean
  - activeTab: 'providers' | 'bulk-update' (default: 'providers')

Flow:
  1. useSession() for auth
  2. If session loading → LoadingSpinner
  3. If unauthorized → redirect(ROUTES.BASE)
  4. useEffect: call fetchHospitalSystemById(id)
     - On success → setHospital(data)
     - On HospitalNotFoundError → setNotFound(true)
     - On other error → setError(message)
  5. If notFound → render 404 view (back link + "Hospital not found" message)
  6. If loading → LoadingSpinner
  7. Render:

Layout (matches Figma):
  <div className={styles.container}>
    <!-- Breadcrumb: chevron + text, above the header card -->
    <Link href={ROUTES.HOSPITAL_SYSTEM_AGREEMENTS} className={styles.breadcrumb}>
      <ChevronLeftIcon /> Hospital System Agreements
    </Link>

    <!-- Header Card: name, metadata, edit button — all inside a Card -->
    <Card>
      <div className={styles.header}>
        <div className={styles.headerInfo}>
          <h1 className={styles.hospitalName}>{hospital.name}</h1>
          <p className={styles.metadata}>
            {hospital.providerCount} Providers · Last Updated: {formatDate(hospital.updatedAt)} by {hospital.updatedBy}
          </p>
        </div>
        <button className={styles.editButton} onClick={handleEditClick}>
          Edit Hospital
        </button>
      </div>
    </Card>

    <!-- Tabs: between header card and content card, NOT inside either -->
    <div className={styles.tabBar}>
      <Tab tabTitle="Providers" isActive={activeTab === 'providers'} onClick={() => setActiveTab('providers')} />
      <Tab tabTitle="Bulk Update" isActive={activeTab === 'bulk-update'} onClick={() => setActiveTab('bulk-update')} />
    </div>

    <!-- Tab Content: inside its own Card -->
    <Card>
      <div className={styles.tabContent}>
        {activeTab === 'providers' && <div className={styles.placeholder}>Providers content — see MLID-1553</div>}
        {activeTab === 'bulk-update' && <div className={styles.placeholder}>Bulk Update content — see D3 tasks</div>}
      </div>
    </Card>
  </div>
```

Key patterns reused from list page:
- `'use client'` directive
- `useSession()` + `canManageHospitalSystems()` access control
- `redirect(ROUTES.BASE)` for unauthorized users
- `LoadingSpinner` for loading state
- `formatDate()` helper (extract from list page into shared util or duplicate locally)
- CSS Module styling consistent with list page design tokens

## Design Decisions

1. **`formatDate` reuse:** The list page has a local `formatDate` function. For now, duplicate it in the detail page. If a third usage appears, extract to a shared util.

2. **Tab content placeholders:** Tabs render but show placeholder text. MLID-1553 will replace the Providers placeholder with the NPI table. D3 tasks will fill the Bulk Update tab.

3. **Edit button handler:** `handleEditClick` will be a no-op or set state for a modal that doesn't exist yet. MLID-1552 will add the `EditHospitalModal` component and wire it.

4. **No server component needed:** Following the list page pattern, this is a client component that fetches data in `useEffect`. This keeps the pattern consistent across the hospital system pages.

5. **Breadcrumb:** Using a `Link` with MUI `ChevronLeftIcon` + text "Hospital System Agreements" (matching Figma). This is a single back-link, not a multi-level breadcrumb, so we use a styled Link rather than the `Breadcrumbs` UI component.

6. **Three-section layout (from Figma):** The page has three distinct visual sections: (1) Header Card with hospital info + edit button, (2) Tab bar sitting between cards (not inside either), (3) Content Card for the active tab's content. This is important — the tabs are NOT inside a Card.
