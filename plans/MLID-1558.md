## Implementation Steps

### Step 0: Security hardening on the API route

**File:** `apps/web/app/api/hospitals/[id]/providers/bulk/route.ts`

Current gaps found:
1. **No NPI trimming** — whitespace-padded NPIs (e.g., `" 1234567890 "`) fail the regex in the service layer instead of being cleaned up. Other routes (e.g., `POST /api/hospitals`) already trim input.
2. **No array size limit** — a client could send millions of entries, causing memory/CPU issues.

Changes:
- Trim each NPI value before passing to the service: `providers.map(p => ({ ...p, npi: p.npi.trim() }))`
- Strip extra fields from provider objects — only keep `npi` (and `name` if present): prevents arbitrary data injection into MongoDB
- Add max array size guard (10,000 entries) with a `400` response: `"providers array exceeds maximum of 10000 entries"`

**Tests** (add new "Input sanitization" describe block to `route.test.ts`):

| Test | Input | Expected |
|------|-------|----------|
| Trims whitespace from NPI values | `{ npi: "  1234567890  " }` | Service receives `{ npi: "1234567890" }` |
| Rejects whitespace-only NPI after trim | `{ npi: "   " }` | 400 — "Each provider must have a valid 'npi' string field" |
| Rejects oversized array | 10,001 entries | 400 — "providers array exceeds maximum of 10000 entries" |
| Strips extra/polluted fields from provider objects | `{ npi: "1234567890", malicious: "payload", __proto__: {} }` | Service receives only `{ npi: "1234567890" }` |
| Rejects NPI with HTML/script injection | `{ npi: "<script>alert(1)</script>" }` | 400 — Invalid NPI format (via service regex) |
| Rejects NPI with null bytes | `{ npi: "12345\x006789" }` | 400 — Invalid NPI format (via service regex) |
| Rejects non-object entries in array | `[null, 42, "string"]` | 400 — "Each provider must have a valid 'npi' string field" |

### Step 1: Add `bulkReplaceProviders` to client service

**File:** `hospitalSystemService.ts`

Add a new exported function following the existing pattern (e.g., `updateHospitalSystem`):

```typescript
export const bulkReplaceProviders = async (
  hospitalId: string,
  npis: string[]
): Promise<HospitalSystemDetail> => {
  const response = await fetch(
    `/api/hospitals/${encodeURIComponent(hospitalId)}/providers/bulk`,
    {
      method: 'PUT',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ providers: npis.map((npi) => ({ npi })) }),
    }
  );

  if (!response.ok) {
    const errorData = await response.json();
    throw new Error(errorData.error || 'Failed to bulk replace providers');
  }

  const result = await response.json();
  return result.data;
};
```

**Tests:** Standard fetch mock tests — success returns data, error throws with API message, network error throws generic message.
