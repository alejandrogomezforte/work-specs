### Data Flow (UI ‚Üí Backend)

```
1. Page mounts ‚Üí useEffect calls loadHospitalSystems(filters)
2. loadHospitalSystems() calls fetchHospitalSystems() (client service)
3. fetchHospitalSystems() builds URLSearchParams ‚Üí fetches GET /api/hospitals
4. GET /api/hospitals (route.ts) ‚Üí authenticates ‚Üí authorizes ‚Üí calls getHospitalSystems()
5. getHospitalSystems() (services/mongodb/hospitalSystem.ts) queries MongoDB
   hospitalSystems collection with pagination, search, sorting
6. Response: API ‚Üí client service ‚Üí page state ‚Üí table renders
```

### URL-Synced State (intakes pattern)

- Filters (`page`, `pageSize`, `sortBy`, `sortOrder`) read from URL on mount via `getFiltersFromUrl()`
- `updateFilters(partial)` merges update ‚Üí sets state + pushes URL + fetches data
- Pagination changes call `updateFilters` which keeps URL in sync

## Implementation Decisions

### Date Format
- Uses a **local `formatDate()` function** (MM/dd/yyyy, UTC) instead of the shared `utils/formatters.ts` `formatDate()` which outputs "Jan 15, 2026"
- UTC methods (`getUTCMonth`, `getUTCDate`, `getUTCFullYear`) to avoid timezone offset shifting dates

### Pagination
- Uses `components/UI/Pagination` (shared, used by 10+ routes including intakes)
- The shared component returns `null` when `totalPages <= 1`
- Added a **fallback row** that always renders "Showing X - Y of Z results" + page size `Select` dropdown when Pagination hides itself
- Page size options: 10, 25, 50, 100 per page
- See `~/Dev/docs/analysis/pagination-refactoring-needs.md` for full pagination component audit

### Table Header Styling
- Background: `#edf4f1` (light teal, matches users-locations pattern)
- Text color: `#134649` (dark teal)
- Font size: `18px`, weight: `500`
- Height: `48px`

### Layout Integration
- Added `/hospital-system-agreements` to `ROUTES_WITHOUT_PATIENT_INFO` in `components/Layout/layout.tsx`
- Added `pathname.startsWith('/hospital-system-agreements')` check for sub-routes
- Without this, the PatientInfo bar was rendering on the page

### MongoDB Collection Name
- Added `collection: COLLECTION.HospitalSystems` to schema options
- Without it, Mongoose auto-lowercased `hospitalSystems` ‚Üí `hospitalsystems`
- Follows same pattern as `DrugConfiguration`, `InsurancePlan`, etc.
