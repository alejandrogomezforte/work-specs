## Implementation Steps

### Step 1 — Pure calculation function (`calculate340B.ts`)

**Files**:
- `apps/web/services/orderEligibility/calculate340B.ts`
- `apps/web/services/orderEligibility/__tests__/calculate340B.test.ts`

**What**: Pure function with zero DB access — fully unit-testable with no mocks.

```typescript
interface Calculate340BInput {
  providerNpi: string | undefined;
  matchingHospitalSystemNames: string[];
  primaryInsurancePayerType: string | null; // null = no primary insurance
}

function calculate340BEligibility(input: Calculate340BInput): string
```

**Logic** (from analysis Eligibility Resolution):
1. If `matchingHospitalSystemNames` is empty → `"No"` (provider not in any hospital system)
2. If `primaryInsurancePayerType` is `null` → `"Waiting on Insurance Input"`
3. If `primaryInsurancePayerType` is `"Commercial"` → join matching names: `"Hospital A, Hospital B"`
4. Otherwise → `"No"`

**TDD tests** (RED first, then implement):
- No matching hospitals → `"No"`
- One matching hospital + `null` insurance → `"Waiting on Insurance Input"`
- One matching hospital + `"Commercial"` → hospital name
- Multiple matching hospitals + `"Commercial"` → comma-joined names
- One matching hospital + `"Medicare"` → `"No"`
- One matching hospital + `"Medicaid"` → `"No"`
- No provider NPI (undefined) → `"No"`

### Step 2 — Data fetchers (`dataFetchers.ts`)

**Files**:
- `apps/web/services/orderEligibility/dataFetchers.ts`
- `apps/web/services/orderEligibility/__tests__/dataFetchers.test.ts`

**What**: Two async functions that handle DB lookups.

**`getHospitalSystemsForNpi(npi: string): Promise<string[]>`**
- Query: `hospitalSystems WHERE deletedAt = null AND 'providers.npi' = npi`
- Returns: array of hospital system names (can be 0, 1, or many)
- Uses Mongoose `HospitalSystem.find()` since model exists

**`getPatientPrimaryInsurance(patientId: string): Promise<{ payerTypeName: string; insurancePlanName: string } | null>`**
- 2-hop lookup:
  1. `patients WHERE _id = ObjectId(patientId)` → get `we_infuse_id_IV`
  2. `patient_insurances WHERE we_infuse_patient_id = we_infuse_id_IV AND priority = '1' AND status = 'Active'`
- Returns: `{ payerTypeName, insurancePlanName }` or `null` if no patient or no primary insurance
- Uses `connectToDatabase()` + raw collection access (patients has no Mongoose model used here; patient_insurances raw for consistency)
- Returns `insurancePlanName` too — not needed for 340B, but D2 (pharmacy) needs it for insurance plan lookup. Avoids re-querying later.

**TDD tests** (mock DB):
- `getHospitalSystemsForNpi`: NPI matches 1 system, matches 2 systems, matches none
- `getPatientPrimaryInsurance`: patient found + insurance found, patient not found, patient found + no insurance, patient has no `we_infuse_id_IV`

### Step 3 — Orchestrator (`orderEligibilityService.ts`)

**Files**:
- `apps/web/services/orderEligibility/orderEligibilityService.ts`
- `apps/web/services/orderEligibility/__tests__/orderEligibilityService.test.ts`

**What**: Ties data fetchers to the pure calculation.

```typescript
async function compute340BForOrder(order: { provider?: string; patient: string }): Promise<string>
```

1. Call `getHospitalSystemsForNpi(order.provider)` → matching names
2. Call `getPatientPrimaryInsurance(order.patient)` → insurance info
3. Call `calculate340BEligibility({ providerNpi, matchingHospitalSystemNames, primaryInsurancePayerType })` → result string
4. Return the result

**TDD tests** (mock data fetchers, not DB):
- Wires fetchers to calculation correctly
- Handles undefined provider (passes through to calculation → `"No"`)
- Returns the calculation result as-is
