## Scope

Reuse the existing **AddNewHospitalModal** by refactoring it into a dual-mode component (`AddEditHospitalModal`) that supports both "add" and "edit" operations. The modal UI is identical between the two modes — only the title, button text, API call, toast messages, and initial input value differ.

### Mode Differences

| Aspect | `add` mode (current) | `edit` mode (new) |
|--------|---------------------|-------------------|
| Title | "Add new Hospital" | "Edit Hospital" |
| Button text | "Add Hospital" / "Adding..." | "Update Hospital" / "Updating..." |
| Initial input | Empty (`""`) | Pre-populated with `initialName` |
| API call | `createHospitalSystem(name)` | `updateHospitalSystem(id, name)` |
| Success toast | "Hospital system added successfully" | "Hospital updated successfully" |
| Error toast | "Failed to add hospital: ..." | "Failed to update hospital: ..." |
| Reset on reopen | Clears to `""` | Resets to `initialName` |

Layout, styling, validation, close behavior, backdrop behavior — all identical.

### Step 1: Add `updateHospitalSystem` to client service (TDD)

**File:** `apps/web/services/hospitalSystems/hospitalSystemService.ts`
**Test file:** `apps/web/services/hospitalSystems/hospitalSystemService.test.ts`

Add a new exported function:

```typescript
export const updateHospitalSystem = async (
  id: string,
  name: string
): Promise<HospitalSystemDetail> => {
  const response = await fetch(`/api/hospitals/${encodeURIComponent(id)}`, {
    method: 'PUT',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ name }),
  });

  if (!response.ok) {
    const errorData = await response.json();
    throw new Error(errorData.error || 'Failed to update hospital system');
  }

  const result = await response.json();
  return result.data;
};
```

- Returns `result.data` on success
