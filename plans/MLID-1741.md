# MLID-1741 - Implement Feature Flag for "Hospital System" Menu Entry

## Overview

Add a `HOSPITAL_SYSTEMS` feature flag that controls visibility of the "Hospital Systems" sidebar nav entry. When the flag is OFF (default), the nav item is hidden. When ON, it renders normally (still gated by `canManageHospitalSystems` permission).

No route/URL blocking is needed â€” hiding the nav entry is sufficient.

## Implementation Details

### 1. Feature Flag Definition (`featureFlags.ts`)

- Add `HOSPITAL_SYSTEMS = 'HOSPITAL_SYSTEMS'` to the `FeatureFlag` enum (after `DRUG_ADMIN_PANEL` line 51)
- Add `[FeatureFlag.HOSPITAL_SYSTEMS]: false` to `DEFAULT_FEATURE_FLAGS` (disabled by default)

### 2. Sidebar Component (`Sidebar.tsx`)

- Add hook: `const isHospitalSystemsEnabled = useFeatureFlag(FeatureFlag.HOSPITAL_SYSTEMS);`
- Modify the Hospital Systems conditional (line 129) to also require the flag:

```typescript
// Before:
if (userObj && canManageHospitalSystems(userObj)) {

// After:
if (isHospitalSystemsEnabled && userObj && canManageHospitalSystems(userObj)) {
```

This means: flag ON **and** user has permission = show link.

- Add 2 new tests:
  1. **Should NOT display Hospital Systems when feature flag is disabled** (admin user with permission but flag OFF)
  2. **Should NOT display Hospital Systems when feature flag is disabled even for Corporate user** (Corporate position but flag OFF)
