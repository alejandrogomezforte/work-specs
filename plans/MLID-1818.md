# D1-T3: 340B Inline Triggers + UI Column (MLID-1818)

## Context

D1-T1 added foundation types (`hospital340BEligible?: string` on `NewOrderDTO`) and the `ORDER_ELIGIBILITY` feature flag. D1-T2 implemented `compute340BForOrder()` which orchestrates hospital NPI lookup + patient insurance lookup + pure calculation to return one of: hospital system name(s), `"Waiting on Insurance Input"`, or `"No"`.

This task wires the computation into the order create/update API and adds a UI column to display the result.

## Implementation

### 1. API Route — POST handler (`route.ts:56-91`)

**Approach:** Compute 340B BEFORE `createOrder()` and set it on the `order` object. Since `createOrder()` spreads `...order` into the MongoDB document (line 946 of `ordersTracker.ts`), the value flows into both main and protocol partner orders within the existing transaction. No extra DB write needed.

```typescript
// After line 70 (order.received = new Date(...)), before createOrder():
try {
  if (await isFeatureEnabled(FeatureFlag.ORDER_ELIGIBILITY)) {
    order.hospital340BEligible = await compute340BForOrder({
      provider: order.provider as string | undefined,
      patient: order.patient as string,
    });
  }
} catch (error) {
  logger.error('Failed to compute 340B eligibility for new order', { error });
}
```

**Imports to add:**
- `import { FeatureFlag, isFeatureEnabled } from '@/utils/featureFlags/featureFlags';`
- `import { compute340BForOrder } from '@/services/orderEligibility';`

### 2. API Route — PUT handler (`route.ts:97-139`)

**Approach:** After building `fieldsToUpdate` and removing undefined fields (line 127), add 340B recomputation if `provider` or `patient` is in the update. The result is added to `fieldsToUpdate` before `updateOrder()`, which propagates to protocol partner via linked order update.

```typescript
// After line 127 (undefined cleanup), before updateOrder():
try {
  if (
    await isFeatureEnabled(FeatureFlag.ORDER_ELIGIBILITY) &&
    (fieldsToUpdate.provider || fieldsToUpdate.patient)
  ) {
    fieldsToUpdate.hospital340BEligible = await compute340BForOrder({
      provider: fieldsToUpdate.provider as string | undefined,
      patient: fieldsToUpdate.patient as string,
    });
  }
} catch (error) {
  logger.error('Failed to compute 340B eligibility for order update', { error });
}
```

**Key details:**
- `fieldsToUpdate.provider` is already normalized to NPI string via `normalizeOrderFieldWith(body.provider, 'npi')` at line 120
- `fieldsToUpdate.patient` is already normalized to ObjectId string via `normalizeOrderFieldWith(body.patient, '_id')` at line 118
- DataGrid always sends the full row, so both fields are present for UI edits
- Error in computation does NOT fail the update (try-catch)
