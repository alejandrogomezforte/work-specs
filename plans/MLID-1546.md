# MLID-1546 - Create API Endpoint for Provider NPI List Retrieval

## Context

This is the first task in Deliverable 2 (Hospital Detail View) of the MLID-1021 epic. The server-side service function `getProviders()` already exists in `services/mongodb/hospitalSystem.ts` (built in MLID-1547). This task creates the API route that exposes it as `GET /api/hospitals/:id/providers`.

**Branch:** `feature/MLID-1546-provider-npi-api` (created from `epic/MLID-1021-hospital-systems-340B`)

## Plan

### 1. Write API route tests (RED) — DONE

**New file:** `apps/web/app/api/hospitals/[id]/providers/route.test.ts`

Follow the exact mock/session pattern from `apps/web/app/api/hospitals/[id]/route.test.ts`.

**Tests (from Jira AC + standard patterns):**
- Returns 401 if not authenticated
- Returns 403 for users without Admin/Auditor/Corporate access
- Returns 400 for invalid ObjectId
- Returns 404 for non-existent hospital (service returns `null`)
- Returns paginated list of NPIs with default params (page=1, pageSize=25)
- Passes custom `page` and `pageSize` query params to service
- Passes `search` query param to filter NPIs
- Returns correct pagination metadata (totalCount, totalPages)

### 2. Create API route handler (GREEN) — DONE

**New file:** `apps/web/app/api/hospitals/[id]/providers/route.ts`

Single `GET` handler following the exact pattern from `[id]/route.ts:27-69`:

1. Auth check → 401
2. Permission check (`canManageHospitalSystems`) → 403
3. DB connection
4. Validate ObjectId → 400
5. Parse query params: `page` (default 1), `pageSize` (default 25), `search` (optional)
6. Call existing `getProviders(id, { page, pageSize, search })`
7. `null` result → 404
8. Return `NextResponse.json(result)` — service already returns `{ data, pagination }` shape

**No service layer changes needed.**

### 3. Refactor + verify (BLUE) — DONE

Tests, types, lint all pass.
