# MLID-1557 ‚Äî Implement NPI Validation Logic

## Context

NPI (National Provider Identifier) validation is currently duplicated in 3 places:
1. `apps/web/models/HospitalSystem.ts` ‚Äî Mongoose schema validator (`NPI_PATTERN`)
2. `apps/web/services/mongodb/hospitalSystem.ts` ‚Äî `bulkUpdateProviders()` inline validation
3. `apps/web/utils/validation/locationValidation.ts` ‚Äî `group_npi` inline check

This task creates a **shared NPI validation utility** that both the CSV upload (MLID-1555) and paste NPI (MLID-1556) features will consume. It also consolidates the duplicated logic into one reusable module.

## Plan

### 1. Create NPI validation utility (TDD)

**File:** `apps/web/utils/validation/npiValidation.ts`
**Test:** `apps/web/utils/validation/__tests__/npiValidation.test.ts`

#### Types to define

```ts
export interface InvalidNpiEntry {
  value: string;    // The raw input that failed
  error: string;    // Human-readable reason
}

export interface NpiValidationResult {
  valid: string[];              // Deduplicated valid NPIs
  invalid: InvalidNpiEntry[];   // Invalid entries with reasons
  totalInput: number;           // Total raw entries received
  duplicatesRemoved: number;    // Count of duplicates stripped
}
```

#### Functions to implement

| Function | Signature | Purpose |
|----------|-----------|---------|
| `isValidNpi` | `(npi: string) => boolean` | Single NPI format check (`/^\d{10}$/`) |
| `validateNpiList` | `(rawEntries: string[]) => NpiValidationResult` | Bulk validation: trim ‚Üí validate ‚Üí dedup ‚Üí return structured result |

#### `validateNpiList` logic

1. Trim whitespace from each entry
2. Filter out empty strings (after trim)
3. Validate each entry against `isValidNpi`
4. Separate into valid/invalid arrays (invalid includes error reason)
5. Deduplicate valid NPIs (keep first occurrence)
6. Return `NpiValidationResult`

#### Error messages

| Condition | Message |
|-----------|---------|
| Empty after trim | `"NPI cannot be empty"` |
| Not 10 characters | `"NPI must be exactly 10 digits (got X characters)"` |
| Non-numeric characters | `"NPI must contain only numeric digits"` |

## Test Cases (RED phase)

1. `isValidNpi("1234567890")` ‚Üí `true`
2. `isValidNpi("123456789")` ‚Üí `false` (9 digits)
3. `isValidNpi("12345678901")` ‚Üí `false` (11 digits)
4. `isValidNpi("123456789a")` ‚Üí `false` (non-numeric)
5. `isValidNpi("")` ‚Üí `false`
6. `isValidNpi("  1234567890  ")` ‚Üí `false` (caller should trim ‚Äî utility is strict)
7. `validateNpiList(["1234567890", "0987654321"])` ‚Üí both valid
8. `validateNpiList(["123", "abcdefghij"])` ‚Üí both invalid with reasons
9. `validateNpiList(["1234567890", "1234567890"])` ‚Üí 1 valid, 1 duplicate removed
10. `validateNpiList(["  1234567890  "])` ‚Üí trims, 1 valid
11. `validateNpiList(["", "  ", "1234567890"])` ‚Üí filters empties, 1 valid
12. `validateNpiList([])` ‚Üí empty result
