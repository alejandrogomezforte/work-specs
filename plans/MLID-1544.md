# MLID-1544: Create API Endpoints for Hospital System CRUD Operations

## Task Summary

Create RESTful API endpoints to manage hospital systems (create, read, update, soft delete). All endpoints require authentication and authorization (Admin role OR Corporate position).

Part of epic **MLID-1021: Referral Tracker - 340B Table for Lookup (Hospital System Agreements)**.

Depends on **MLID-1543** (HospitalSystem model) — already implemented on parent branch.

| Method | Route | Description | Auth |
|--------|-------|-------------|------|
| `GET` | `/api/hospitals` | List with pagination, search, sorting | Admin / Corporate |
| `GET` | `/api/hospitals/:id` | Get single hospital with providers | Admin / Corporate |
| `POST` | `/api/hospitals` | Create new hospital | Admin / Corporate |
| `PUT` | `/api/hospitals/:id` | Update hospital name | Admin / Corporate |

#### Service Interface

```typescript
interface GetHospitalSystemsParams {
  search?: string;
  page: number;
  pageSize: number;
  sortBy?: 'name' | 'updatedAt' | 'providerCount';
  sortOrder?: 'asc' | 'desc';
}

interface GetHospitalSystemsResult {
  hospitalSystems: HospitalSystemListItem[];
  totalCount: number;
  totalPages: number;
}

interface HospitalSystemListItem {
  _id: string;
  name: string;
  providerCount: number;
  updatedAt: Date;
  updatedBy?: string;
}
```

#### Service Functions

1. **`getHospitalSystems(params)`** — Paginated list
   - Filter: `{ deletedAt: null }` (exclude soft-deleted)
   - Search: regex on `name` field (using `escapeRegex` for safety)
   - Sort: configurable field + direction (default: `name` asc)
   - Pagination: `skip`/`limit` pattern
   - Return: list items with `providerCount` (from `providerNPIs.length`), `updatedAt`, `updatedBy`

2. **`getHospitalSystemById(id)`** — Single record
   - Filter: `{ _id: id, deletedAt: null }`
   - Return full document or `null`

3. **`createHospitalSystem(data, userEmail)`** — Create
   - Set `createdBy` and `updatedBy` to `userEmail`
   - Return created document

4. **`updateHospitalSystem(id, data, userEmail)`** — Update
   - Filter: `{ _id: id, deletedAt: null }` (can't update soft-deleted)
   - Set `updatedBy` to `userEmail`
   - Use `findOneAndUpdate` with `{ new: true }`
   - Return updated document or `null`

5. **`deleteHospitalSystem(id, userEmail)`** — Soft delete
   - Set `deletedAt: new Date()` and `updatedBy: userEmail`
   - Filter: `{ _id: id, deletedAt: null }` (can't delete already-deleted)
   - Return updated document or `null`

## Notes: Pagination Standard for New API Endpoints

### Standard Pagination Response Format

All new paginated API endpoints should return this response shape:

```json
{
  "data": [ ... ],
  "pagination": {
    "page": 1,
    "pageSize": 25,
    "totalCount": 42,
    "totalPages": 2
  }
}
```

Some older admin endpoints also include `"success": true` at the top level. The locations endpoint (newest reference) omits it and returns `{ data, pagination }` directly — the service result is passed through to the response as-is.

### Standard Query Parameters

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `page` | number | `1` | Page number (1-indexed) |
| `pageSize` | number | `25` | Items per page |
| `search` | string | `''` | Search term (case-insensitive regex) |
| `sortBy` | string | varies | Field to sort by |
| `sortOrder` | `'asc'` \| `'desc'` | `'asc'` | Sort direction |

### Endpoints Using This Pagination Pattern

| Endpoint | Route File | Service File | Notes |
|----------|-----------|--------------|-------|
| **Locations** | `apps/web/app/api/admin/locations/route.ts` | `apps/web/services/locations/locationDbService.ts` | Newest reference. Returns `{ data, pagination }` directly. Supports `sortBy`, `sortOrder`, `search`, `type` filter. |
| **Biosimilars** | `apps/web/app/api/admin/biosimilars/route.ts` | `apps/web/services/mongodb/biosimilars.ts` | Returns `{ success, data, pagination }`. Uses `validatePaginationParams()`. |
| **LI Drugs** | `apps/web/app/api/admin/lidrugs/route.ts` | `apps/web/services/mongodb/liDrugs.ts` | Returns `{ success, data, pagination }`. Uses `validatePaginationParams()`. |
| **Drug Settings** | `apps/web/app/api/admin/drug-settings/route.ts` | Inline in route (uses `DrugSettings` model directly) | Returns `{ success, data, pagination }`. Builds pagination inline. |

### Example Code: Locations Endpoint (Recommended Pattern)

**Route** — `apps/web/app/api/admin/locations/route.ts`

```typescript
export async function GET(request: NextRequest): Promise<NextResponse> {
  try {
    const session = await getServerSession(authOptions);
    if (!session?.user) {
      return NextResponse.json({ error: 'Unauthorized' }, { status: 401 });
    }

    const { searchParams } = new URL(request.url);

    const params: LocationsListParams = {
      page: parseInt(searchParams.get('page') || '1'),
      pageSize: parseInt(searchParams.get('pageSize') || '25'),
      search: searchParams.get('search') || '',
      sortBy: searchParams.get('sortBy') || 'facility_name',
      sortOrder: (searchParams.get('sortOrder') as 'asc' | 'desc') || 'asc',
    };

    const result = await fetchLocationsFromDB(params);

    return NextResponse.json(result);
    // result shape: { data: [...], pagination: { page, pageSize, totalCount, totalPages } }
  } catch (error) {
    // ...
  }
}
```

**Service** — `apps/web/services/locations/locationDbService.ts`

```typescript
export interface LocationsListParams {
  page?: number;
  pageSize?: number;
  search?: string;
  sortBy?: string;
  sortOrder?: 'asc' | 'desc';
}

export interface LocationsListResponse {
  data: ILocation[];
  pagination: {
    page: number;
    pageSize: number;
    totalCount: number;
    totalPages: number;
  };
}

export const fetchLocationsFromDB = async (
  params: LocationsListParams = {}
): Promise<LocationsListResponse> => {
  const {
    page = 1,
    pageSize = 25,
    search = '',
    sortBy = 'facility_name',
    sortOrder = 'asc',
  } = params;

  // Build search query
  const searchQuery = search
    ? {
        $or: [
          { location_name: { $regex: search, $options: 'i' } },
          { facility_name: { $regex: search, $options: 'i' } },
        ],
      }
    : {};

  const query = { ...searchQuery };
  const sortQuery = { [sortBy]: sortOrder === 'asc' ? 1 : -1 };

  const totalCount = await collection.countDocuments(query);
  const totalPages = Math.ceil(totalCount / pageSize);
  const skip = (page - 1) * pageSize;

  const data = await collection
    .find(query)
    .sort(sortQuery)
    .skip(skip)
    .limit(pageSize)
    .toArray();

  return {
    data,
    pagination: { page, pageSize, totalCount, totalPages },
  };
};
```

### Example Code: Biosimilars Endpoint (Alternate Pattern with `success`)

**Route** — `apps/web/app/api/admin/biosimilars/route.ts`

```typescript
export async function GET(request: Request) {
  try {
    const session = await getServerSession(authOptions);
    if (!session || !session.user) {
      return NextResponse.json({ error: 'Unauthorized' }, { status: 401 });
    }

    await connectToDatabase();

    const { searchParams } = new URL(request.url);
    const rawSearch = searchParams.get('search') || undefined;
    const search =
      rawSearch && rawSearch.length > MAX_SEARCH_LENGTH
        ? rawSearch.substring(0, MAX_SEARCH_LENGTH)
        : rawSearch;

    const validationResult = validatePaginationParams(
      searchParams.get('page'),
      searchParams.get('pageSize')
    );

    if (!validationResult.isValid) {
      return NextResponse.json(
        { error: validationResult.error },
        { status: 400 }
      );
    }

    const { page, pageSize } = validationResult.params!;
    const result = await getBiosimilars({ search, page, pageSize });

    return NextResponse.json({
      success: true,
      data: result.biosimilars,
      pagination: {
        page,
        pageSize,
        totalCount: result.totalCount,
        totalPages: result.totalPages,
      },
    });
  } catch (error) {
    // ...
  }
}
```

### Example Code: Drug Settings (Inline Pagination, No Service Layer)

**Route** — `apps/web/app/api/admin/drug-settings/route.ts`

```typescript
export async function GET(request: NextRequest) {
  // ... auth + authz checks ...

  const { searchParams } = new URL(request.url);
  const search = searchParams.get('search');
  const page = parseInt(searchParams.get('page') || '1', 10);
  const pageSize = parseInt(searchParams.get('pageSize') || '25', 10);

  const query: DrugSettingsQuery = {};
  if (search) {
    query.name = { $regex: escapeRegex(search), $options: 'i' };
  }

  const totalCount = await DrugSettings.countDocuments(query);
  const settings = await DrugSettings.find(query)
    .sort({ name: 1 })
    .skip((page - 1) * pageSize)
    .limit(pageSize)
    .lean();

  return NextResponse.json({
    success: true,
    data: settings,
    pagination: {
      page,
      pageSize,
      totalCount,
      totalPages: Math.ceil(totalCount / pageSize),
    },
  });
}
```

### Key Takeaways

1. **Response shape is consistent**: `{ data, pagination: { page, pageSize, totalCount, totalPages } }`
2. **Query params are consistent**: `page`, `pageSize`, `search`, `sortBy`, `sortOrder`
3. **Defaults**: `page = 1`, `pageSize = 25`, `sortOrder = 'asc'`
4. **Pagination math**: `skip = (page - 1) * pageSize`, `totalPages = Math.ceil(totalCount / pageSize)`
5. **Search**: always case-insensitive regex, always escape user input with `escapeRegex()`
6. **The locations endpoint is the newest and cleanest pattern** — service returns `{ data, pagination }` directly, route passes it through
