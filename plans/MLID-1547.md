### Function specifications

#### `getProviders(hospitalId, options)`

- Retrieves a hospital system by ID (must not be soft-deleted)
- Returns paginated list of NPIs from the `providerNPIs` array
- Supports search by NPI substring
- Returns `{ data: string[], pagination: { page, pageSize, totalCount, totalPages } }`
- Returns `null` if hospital not found

**Pagination approach:** Since NPIs are stored as an array field on the document (not a separate collection), pagination is done in-memory after fetching the document. This is acceptable because NPI lists are bounded in size (hundreds, not millions).

#### `bulkUpdateProviders(hospitalId, npis, userId)`

- Validates each NPI: exactly 10 digits, numeric only
- Returns validation errors with the specific invalid NPIs
- Replaces the entire `providerNPIs` array (not append)
- Deduplicates NPIs before saving
- Updates `updatedBy` field with `userId`
- Returns `null` if hospital not found (soft-deleted or non-existent)
- Returns the updated hospital document on success

**NPI validation:** Reuses the same `^\d{10}$` pattern from the HospitalSystem model schema.
