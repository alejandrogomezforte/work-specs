# MLID-1556 — Implement Paste NPI List Functionality

## Context

Most of the paste NPI functionality was already implemented in MLID-1555 (CSV upload task) — the textarea, live validation, validation summary with invalid/duplicate details, and Confirm & Replace button enabling. This task closes the remaining gaps identified by comparing the Jira acceptance criteria against the current implementation on the epic branch.

---

## Gaps to Close

### 1. MUI component for textarea

**Jira says:** "Use existing Textarea/Input component from `components/UI/MUI`"

**Current state:** Plain `<textarea>` element with CSS module styling.

**Decision:** The only MUI textarea wrapper is `MuiTextField` which requires `react-hook-form` (`control`/`watch` props). The BulkUpdateTab doesn't use react-hook-form — adding it just for a textarea would be over-engineering. Instead, use a plain MUI `TextField` with `multiline` prop directly. This gives MUI styling consistency without the form dependency.

### 2. Label text

**Jira says:** Textarea labeled "Paste NPI List (from Excel)"

**Current state:** Label is "Paste NPI List"

**Fix:** Change `sectionTitle` text to "Paste NPI List (from Excel)".

### 3. CR line ending support

**Jira says:** Handles various line endings (Windows, Unix, Mac)

**Current state:** `handlePasteChange` splits by `\n` only. CRLF (`\r\n`) works because `\n` is present. But standalone CR (`\r`, old Mac) is not handled.

**Fix:** Update split regex in `handlePasteChange` from `\n` to `/\r\n|\r|\n/`.

### 4. Missing unit tests

Add tests for:
- Textarea renders with correct label ("Paste NPI List (from Excel)")
- Handles Windows line endings (CRLF `\r\n`)
- Handles Mac line endings (CR `\r`)
- Pasting new text replaces previous validation (not append)
