# MLID-1543: Create MongoDB Schema for Hospital System Agreements

## Task Summary

Create a `HospitalSystem` Mongoose model to store hospital system agreements and their associated provider NPIs. This is part of the **Referral Tracker - 340B Table for Lookup (Hospital System Agreements)** epic (MLID-1021).

## Acceptance Criteria (from Jira)

- `HospitalSystem` model with fields: `_id`, `name`, `providerNPIs`, `createdAt`, `updatedAt`, `updatedBy`, `createdBy`, `deletedAt`
- Text index on `name` for search
- NPI validated as 10-digit numeric string
- Virtual for provider count
- An NPI can belong to multiple hospital systems (many-to-many)
- Unit tests for validation, virtual, and indexes

#### 2b: Implement the Model (Green Phase)

**File:** `apps/web/models/HospitalSystem.ts`

```typescript
// Interface
export interface HospitalSystemDocument extends Document {
  name: string;
  providerNPIs: string[];
  createdBy?: string;
  updatedBy?: string;
  deletedAt?: Date | null;
  createdAt?: Date;
  updatedAt?: Date;
  providerCount: number; // virtual
}

// Schema
const HospitalSystemSchema = new Schema<HospitalSystemDocument>(
  {
    name: { type: String, required: true },
    providerNPIs: {
      type: [String],
      default: [],
      validate: {
        validator: (npis: string[]) => npis.every(npi => /^\d{10}$/.test(npi)),
        message: 'Each NPI must be a 10-digit numeric string'
      }
    },
    createdBy: { type: String },
    updatedBy: { type: String },
    deletedAt: { type: Date, default: null },
  },
  { timestamps: true, toJSON: { virtuals: true }, toObject: { virtuals: true } }
);

// Virtual: providerCount
HospitalSystemSchema.virtual('providerCount').get(function () {
  return this.providerNPIs?.length ?? 0;
});

// Indexes
HospitalSystemSchema.index({ name: 'text' });
HospitalSystemSchema.index({ deletedAt: 1 });
```

**Key design decisions:**
- **NPI validation via array-level validator:** Validates each element matches `/^\d{10}$/`. This is the Mongoose-idiomatic approach for array element validation.
- **`deletedAt` for soft delete:** Nullable date field. `null` = active, date = soft-deleted. Index on `deletedAt` for efficient querying of active records.
- **Virtual `providerCount`:** Computed from `providerNPIs.length`. Enabled via `toJSON: { virtuals: true }` and `toObject: { virtuals: true }`.
- **Many-to-many NPI relationship:** The same NPI string can appear in multiple HospitalSystem documents. No unique constraint on NPIs across documents.
- **Model registration pattern:** Follows the existing Next.js hot-reload safe pattern: `mongoose.models[modelName] || mongoose.model(...)`.
- **No migration needed:** New collection, Mongoose auto-creates collection and indexes on first use (`autoIndex: true` in connect.ts).
