### Pattern to follow

The **AddNewDrugModal** (`components/Modules/drug-admin-panel/components/AddNewDrugModal.tsx`) is the closest existing pattern. Key elements:

- MUI `Dialog`, `DialogTitle`, `DialogContent`, `DialogActions`
- `CloseIcon` IconButton in title
- MUI `TextField` for input
- Custom styled Cancel / Submit buttons (CSS Module)
- `showToast()` for success/error feedback
- Backdrop click prevention (`handleDialogClose`)
- Form reset on modal close via `useEffect`

### API contract

**POST /api/hospitals**

Request: `{ name: string }`

Responses:
- `201` — `{ data: HospitalSystemDocument }`
- `400` — `{ error: "Hospital system name is required" }`
- `409` — `{ error: "Hospital system with this name already exists" }`
- `401` / `403` — auth errors (handled by page-level access control)

## Implementation Plan

### Step 1: Add `createHospitalSystem()` to client service

**File:** `apps/web/services/hospitalSystems/hospitalSystemService.ts`

Add a new exported function:

```ts
export const createHospitalSystem = async (name: string): Promise<HospitalSystemListItem> => {
  const response = await fetch('/api/hospitals', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ name }),
  });

  if (!response.ok) {
    const errorData = await response.json();
    throw new Error(errorData.error || 'Failed to create hospital system');
  }

  const result = await response.json();
  return result.data;
};
```

### Step 2: Create AddNewHospitalModal component (TDD)

**Location:** `apps/web/app/hospital-system-agreements/components/AddNewHospitalModal.tsx`

**Props:**
```ts
interface AddNewHospitalModalProps {
  isOpen: boolean;
  onClose: () => void;
  onSuccess: () => void;
}
```

**Component structure (per design mockups):**
- MUI `Dialog` with `maxWidth="sm"`, `fullWidth`
- `DialogTitle` with "Add new Hospital" title + Close (X) IconButton
- `DialogContent` with:
  - Label: "Hospital name" (Typography)
  - MUI `TextField` — placeholder: "Enter Hospital name"
  - Required field
  - `error` + `helperText` for inline validation
  - "Add Hospital" submit button below the input (right-aligned)
- **No Cancel button** — only X close and submit (per design)
- **No `DialogActions`** — submit button is inside `DialogContent`
- Backdrop click prevention
- Form reset on close/reopen

**State management:**
- `hospitalName` (string)
- `error` (string | null) — inline validation error
- `isSubmitting` (boolean) — loading state

**Behavior:**
- Validate: empty/whitespace-only name → show "Hospital name is required"
- Clear error when user types
- On submit: call `createHospitalSystem(name.trim())`
  - Success: `showToast` success, call `onSuccess()`, call `onClose()`
  - 409 (duplicate): show inline error "A hospital system with this name already exists"
  - Other error: show inline error + error toast
- Keep modal open on error
- Disable submit button when: name is empty, or submitting
- Show "Adding..." text on submit button while submitting
