# Personal Engineering Specs

Personal knowledge base for engineering work on the Local Infusion platform. Contains plans, analysis, PR documentation, and development workflows.

## Structure

```
.
├── plans/       # Implementation plans for tasks and epics
├── analysis/    # Data analysis, architecture research, and technical deep-dives
├── PR/          # Pull request descriptions documenting completed work
└── dx/          # Developer experience — workflows, branching strategy, problem-solving approach
```

### plans/

Implementation plans crafted before writing code. Each plan breaks down a task or epic into concrete steps, captures design decisions, and tracks progress. Plans are the blueprint — the PR is the receipt.

### analysis/

Technical investigation documents. Data shape explorations, architecture breakdowns, feasibility studies, and any research needed to make informed decisions before planning begins.

### PR/

Pull request descriptions that document the "what and why" of completed work. Serves as a historical record of delivered features, bug fixes, and refactors across epics and individual tasks.

### dx/

Personal development methodology. Git branching model, workflow rules, and the structured approach used to attack any problem from discovery through delivery.
