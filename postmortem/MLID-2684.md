# MLID-2684 — Prior Auth PDF preview shows full fax vs parsed document

Bug (High). Reporter: Olha Turovych. Assignee: Alejandro Gomez.

## Ticket summary

Two screens preview what QA believes is "the same prior-auth document", but the
previews differ:

- **Patient card → Prior Auth table** (`/patient/[id]/prior-auth`): the preview
  shows **the whole fax** (all pages).
- **Order Details → Documents tab** (`/orders-tracker/new/[id]/documents`): the
  preview shows **only the parsed/split page**.

**Expected (per QA):** both previews should show only the parsed prior-auth
document.

**Actual:** patient card shows the full fax; order details shows a single parsed
page.

## Root cause / investigation finding

The two screens are **not showing the same object**. They read from two
different collections and preview two different blobs:

| | Patient Prior Auth table | Order Details → Documents tab |
|---|---|---|
| API route | `app/api/patient/[patientId]/prior-auth/route.ts` | `app/api/orders/details/route.ts` |
| Collection | **`intakes`** | **`documents`** |
| Match | `patient.patientId == "10043"` (we_infuse id) | `patientId == "6a424db6febab4f6ff88df72"` (mongo id) |
| One row represents | the **whole intake / fax** (parent) | one **split page** of the intake (child) |
| Preview URL source | `intake.documents[0].pdfDocument` → `Fax.pdf` (full fax) | `document.blobPath` → single split-page PDF |
| Transform | `transformIntakeToPriorAuthLetter` | `orders/details` route inline `.map` |

An **intake** holds the FULL PDF (every page the doctor faxed). During page
splitting, each parsed section becomes its own row in the **documents**
collection with its own `blobPath`. So the prior-auth table (intakes) previews
the full fax, while the documents tab (documents) previews an individual split
page. QA is comparing a parent intake against a child split document.

### Secondary observations
- The `Referral Copy` split document (`6a424dbafebab4f6ff88df85`) also points at
  the full `Fax.pdf` (`isFullDocument: true`), so the whole fax is reachable from
  the Documents tab too — just under a different row.
- `transformIntakeToPriorAuthLetter` sets `documentPage` from a
  `documentDefinitions` entry of type `'prior-authorization'`, but this intake's
  definition is typed `'Insurance Authorization'`. They never match, so
  `documentPage` is `undefined` and the viewer cannot jump to the auth page.
- This intake is not actually a prior-auth: `documentType.value == "Other"`,
  `documentTypeByAi.value == "Fax cover sheet / transmittal page"`. It appears in
  the prior-auth table only because the base match requires
  `letterMetadata.documentType.value` to *exist* (it is "Other"), not to be a
  real prior-auth.

## Conclusion

Theory confirmed: the two surfaces render objects from different collections —
`intakes` (full fax PDF) on the patient Prior Auth table vs `documents` (split
pages) on the Order Details Documents tab. They are parent-vs-child, not the same
document, so the differing previews are expected given the current data model.
